# TIL 저장 자동화

**트리거**: "TIL 저장해줘" / "오늘 내용 정리해줘" / "블로그 정리해줘"

## 동작 순서
1. 현재 대화 주제를 파악
2. `hancom/hancom/TIL/` 안의 기존 파일들을 확인
3. 밀접하게 연관된 파일이 있으면 → 기존 파일 수정 보완
   연관 파일 없으면 → 새 파일 생성 (아래 규칙대로)
4. `hancom/hancom/TIL/` 폴더가 없으면 생성

## 파일명 규칙
- 형식: `YYMMDD_과목_번호_주제.md`
- 예시: `260702_css_12_neon.md`

## 저장 경로
`hancom/hancom/TIL/` 고정
- 과목이 `claude` → `hancom/hancom/TIL/claude/` 하위 폴더에 저장
- 과목이 강의 내용(css, web, tool, js 등 실습/강의 관련) → `hancom/hancom/TIL/강의/` 하위 폴더에 저장

## 파일 내용 구조
```markdown
# [과목 번호] 주제

> 작성: YYYY-MM-DD

## Q&A

**Q. 질문 (대화에서 다듬어 작성)**

답변 (핵심만 간결하게)

**Q. 다음 질문**

답변

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

## 개념 요약

핵심 개념들을 항목별로 정리.
표, 코드블록 적극 활용.
개념+코드만 나열하면 이해가 안 되므로, 각 항목마다 **실무에서 어떤 상황/방식으로 쓰이는지 예시를 반드시 함께 설명**.

| 개념 | 설명 |
|---|---|
| 항목 | 내용 |

```css
/* 코드 예시 */
```

**실무 예시**: 이 개념이 실제 서비스/프로젝트에서 어떤 상황에 쓰이는지 한두 문장으로.
```

**형식 원칙**: Q&A로 맥락을 살리되, 하단에 개념 요약도 함께 정리.
질문은 대화 그대로 옮기지 않고, 나중에 다시 봤을 때 맥락 없이도 이해할 수 있는 수준으로 재구성.
짧고 단편적인 질문은 "무엇이 왜 다른가", "어떤 상황에서 쓰는가" 식으로 풀어서 작성.
Q&A 순서는 대화가 오간 시간순이 아니라 **개념적으로 알면 좋은 순서(기초 → 응용)**로 재배열. 예: JSX 기초 표현식 → 리스트/폼 → UI 라이브러리 조립 → 레이아웃 → useEffect → fetch처럼 배우는 사람이 자연스럽게 따라올 수 있는 순서로.
개념 요약의 각 항목은 개념 설명 + 코드로 끝내지 말고, "실무에서는 이런 상황에 이렇게 쓰인다"는 예시를 항상 덧붙일 것 — 예시 없이 개념/코드만 있으면 이해가 안 된다는 피드백.
Velog 등 마크다운 블로그에 바로 붙여넣을 수 있는 수준으로 작성.
섹션 구분선은 `---` 대신 `━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━`(유니코드 굵은 선, 40자) 사용 — Velog 테마에서 `---`(hr)는 옅은 회색으로만 나오기 때문.

## 기존 파일 수정 보완 규칙
- 연관 여부는 Claude가 내용 기반으로 판단
- 수정 시 헤더 날짜를 `> 작성: YYYY-MM-DD | 수정: YYYY-MM-DD` 형식으로 변경
- 새 내용은 파일 맨 하단에 섹션으로 추가
- 추가된 섹션 앞에 `## [수정: YYYY-MM-DD] 추가 개념명` 표기

```markdown
> 작성: 2026-07-02 | 수정: 2026-07-10

...기존 내용...

## [수정: 2026-07-10] 추가 개념명

추가 내용
```
