HTML CSS 정리

HTML

HTML이란?

HTML(HyperText Markup Language)은 웹 페이지의 뼈대(구조)를 만드는 마크업 언어입니다. "이 텍스트는 제목이다", "이건 표다", "이건 입력칸이다" 처럼 콘텐츠의 의미와 구조를 태그로 표시하며, 실제 디자인(색, 크기, 배치)은 CSS가 담당합니다.

1. 기본 구조 (Boilerplate)

모든 HTML 문서에 공통으로 들어가는 뼈대입니다.

태그역할
<!DOCTYPE html>"이 문서는 HTML5 문서다" 선언
<html lang="ko">문서 시작, 언어 설정
<head>화면에 안 보이는 설정 칸 (메타정보, 제목, CSS 연결)
<meta charset="UTF-8">한글이 깨지지 않게 인코딩 지정
<title>브라우저 탭에 뜨는 제목
<body>화면에 실제로 보이는 내용 칸
<!DOCTYPE html>
<html lang="ko">
  <head>
    <meta charset="UTF-8">
    <title>자 드가자</title>
  </head>
  <body>
    <h1>안녕하세요! 제 첫 웹페이지입니다.</h1>
  </body>
</html>

7. 시맨틱 레이아웃 — div/span과의 차이

div는 뜻이 없는 빈 상자(블록)이고, span은 뜻이 없는 빈 상자(인라인, 줄바꿈 없음)입니다. 반면 시맨틱 태그는 이름만 봐도 어떤 구역인지 드러납니다.

시맨틱 태그역할
<header>페이지/섹션 상단 (제목, 네비게이션)
<nav>내비게이션 메뉴
<main>페이지의 핵심 콘텐츠
<section>주제로 묶인 구역
<article>독립적으로 완결된 글 하나
<aside>본문 옆 보조 정보 (사이드바)
<footer>페이지/섹션 하단
<div>뜻 없는 블록 상자 — 스타일링을 위한 그룹핑용
<span>뜻 없는 인라인 상자 — 문장 중 일부만 감쌀 때
<header>
  <h1>시맨틱 구조</h1>
  <nav>홈 ㅣ 소개 ㅣ 연락처</nav>
</header>
<main>
  <section>
    <article>첫 번째 글 내용</article>
    <article>두 번째 글 내용</article>
  </section>
</main>
<aside>곁다리 정보(사이드바)</aside>
<footer>2026 내 사이트</footer>

핵심: header, nav, main, section, article, aside, footer

<div class="card"> <!-- div: 뜻 없는 구역 나누기용 상자 -->
  <img src="...">
  <h3>운동화</h3>
  <p><strong>99,999원</strong></p>
</div>
<p>글자 중 <span>여기만</span> 강조</p> <!-- span: 줄 안 바꾸는 인라인 상자 -->

언제 무엇을 쓸까?
- 페이지의 큰 구역(머리말/메뉴/본문/사이드바/꼬리말)을 나눌 때 → 뜻이 맞는 시맨틱 태그
- 스타일링만을 위해 여러 요소를 감싸야 하는데 마땅한 의미가 없을 때 → div(블록) / span(인라인)

CSS

CSS란?

CSS(Cascading Style Sheets)는 HTML로 만든 뼈대에 디자인(색, 크기, 배치, 애니메이션 등)을 입히는 언어입니다. "어떤 요소를(선택자) 어떻게 꾸밀지(속성: 값)"를 규칙으로 작성합니다.

1. 선택자 (Selector)

선택자의미예
타입 선택자태그 이름으로 선택h1 { }
클래스 선택자class="..." 로 선택.special { }
ID 선택자id="..." 로 선택 (문서 내 유일)#li1 { }
자손 선택자A 안의 Bli em { }
:hover 등 pseudo-class특정 상태일 때a:hover { }
::before/::after요소 앞/뒤에 가상 콘텐츠 삽입.info::after { }
/* 우선순위(specificity): 타입 < 클래스 < 자손 조합 < 인라인 순으로 점점 구체적 */
h1 { color: red }
p { color: green }
.special { color: orange }
li em { color: purple }

2. 박스모델 (Box Model)

속성역할
border테두리 (두께, 스타일, 색)
margin요소 바깥 여백
padding요소 안쪽 여백(테두리와 내용 사이)
box-sizing: border-boxwidth/height에 border·padding까지 포함해서 계산
border-radius모서리 둥글게
list-style: none리스트 기본 불릿 제거
ul {
  background-color: white;
  border: 6px double navy;
  height: 100px;
  list-style: none;
  margin: 20px;
  padding: 30px;
  width: 100px;
}

3. Flexbox

속성역할
display: flex자식 요소들을 한 줄(가로/세로)로 배치
flex-direction배치 방향 (row/column)
align-items세로축 정렬
justify-content가로축 정렬
gap요소 사이 간격
flex: 1남는 공간을 균등하게 나눠 가짐
.box {
  display: flex;
  flex-direction: row; /* 가로 배치: 왼 -> 오 */
  gap: 10px;
}
.card {
  flex: 1; /* 남는 가로 공간 균등 분할 → 카드 너비 동일 */
}

4. Grid

속성역할
display: grid2차원 격자 레이아웃 켜기
grid-template-columns열 개수/너비 지정 (repeat(2, 1fr) = 동일 너비 2열)
gap격자 칸 사이 간격
.grid {
  display: grid;
  grid-template-columns: repeat(2, 1fr);
  gap: 12px;
}

5. Position (배치)

값역할
relative원래 자리를 기준으로 살짝 이동 (자기 자리는 그대로 차지)
absolute가장 가까운 relative 조상 기준으로 배치
fixed화면(뷰포트) 기준 고정, 스크롤해도 안 움직임
sticky스크롤하다 특정 지점에서 딱 붙음
z-index겹칠 때 위/아래 순서
.abs {
  position: absolute;
  bottom: 10px;
  right: 16px;
  z-index: 10;
}
.fixed {
  position: fixed;
  bottom: 16px;
  right: 16px;
}
.sticky {
  position: sticky;
  top: 0;
}

6. Transform / Transition / Animation

속성역할
transform이동(translate)/회전(rotate)/크기(scale) 변형
transition속성 값이 바뀔 때 부드럽게 변화
@keyframes + animation반복/자동 애니메이션 정의
@keyframes float {
  0%, 100% { transform: translateY(0); }
  50% { transform: translateY(-30px); }
}
.ball:hover {
  animation: float 1.5s ease-in-out infinite;
}
.card {
  opacity: 0.85;
  transition: all 0.3s ease; /* 바뀔 때 0.3초 부드럽게 */
}
.card:hover {
  opacity: 1;
  transform: translateY(-8px);
  box-shadow: 0 16px 32px rgb(115, 138, 80);
}

7. 시각 효과 (그림자 / 그라데이션 / 가상요소 꾸미기)

.info { position: relative; }
.info::after {
  content: "도움말";
  display: none;
}
.info:hover::after {
  display: block;
  position: absolute;
  background: black;
  color: white;
  border-radius: 4px;
}

순수 CSS만으로 만드는 툴팁(hover 시 말풍선)

.quote::before { content: "'"; color: tomato; }
.quote::after  { content: "'"; color: tomato; }

::before/::after의 content로 텍스트 앞뒤에 장식 추가

8. 반응형 (Media Query)

/* 화면 폭이 600px 이하일 때만 아래 규칙 적용 (모바일) */
@media (max-width: 600px) {
  .box {
    flex-direction: column;
  }
}

9. CSS 변수 (Custom Properties)

:root {
  --main-color: red;
  --gap: 12px;
}
.row {
  display: flex;
  gap: var(--gap);
}
.btn {
  background: var(--main-color);
}

방법작성 위치우선순위

...
<!-- 인라인 -->

11. CSS Reset

브라우저마다 다른 기본 여백/스타일을 없애고 동일한 출발선에서 시작하기 위한 초기화 패턴입니다.

* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}
QUIZ

1

[HTML 1] 시맨틱 태그

다음 코드는 실제로 자주 보이는 "div로만 만든" 페이지 구조입니다. 실무에서 SEO와 접근성(스크린리더) 때문에 아래 코드를 시맨틱하게 바꾼다면, 각 div를 어떤 태그로 바꾸는 것이 가장 적절할까요?

<div class="top">로고 / 메뉴</div>
<div class="content">
  <div class="post">글 하나</div>
</div>
<div class="side">광고, 관련 글</div>
<div class="bottom">저작권 정보</div>

A) 전부 <section>으로 통일한다
B) top → <header>, content → <main>, post → <article>, side → <aside>, bottom → <footer>
C) 전부 <div>를 유지하고 class 이름만 명확히 짓는다
D) top → <nav>, content → <div>, side → <div>, bottom → <div>

정답: B
header(상단 로고/메뉴), main(핵심 콘텐츠), article(독립된 글), aside(사이드바), footer(하단 정보)로 나누면 이름만 봐도 구조를 알 수 있어 SEO와 스크린리더 탐색에 유리합니다.

2

[HTML 2] 폼(form) 접근성 / 유효성 검사

다음은 실무 회원가입 폼에서 흔히 발견되는 문제 코드입니다.

<p>이메일</p>
<input type="text" placeholder="이메일 입력">
<button>가입하기</button>

이 코드를 실무 기준(접근성 + 기본 유효성 검사)으로 개선하려면 무엇을 고쳐야 할까요? (가장 핵심적인 것 하나를 고르세요)

A) placeholder 문구를 더 친절하게 바꾼다
B) <p>를 <label for="email">로 바꾸고 <input id="email" type="email" required>로 연결/타입 지정한다
C) <button>을 <a>로 바꾼다
D) <input>에 style="color:red"를 추가한다

정답: B
<label for>와 <input id>를 짝지어야 스크린리더가 "이 입력칸이 이메일 칸"이라는 걸 인식하고, type="email"과 required는 브라우저가 자체적으로 형식/필수 여부를 검사해줍니다.

3

[CSS 1] Flexbox

카드 3개를 가로로 나열하면서 "화면 너비가 얼마든 3개가 항상 동일한 너비"가 되게 만들고 싶습니다. 부모에 display:flex를 준 상태에서, 각 카드(.card)에 추가해야 할 속성은?

.row { display: flex; gap: 10px; }
.card { /* 여기에 무엇을 추가? */ }

A) .card { width: auto; }
B) .card { flex: 1; }
C) .card { position: absolute; }
D) .card { display: block; }

정답: B
flex: 1은 "남는 공간을 균등하게 나눠 가진다"는 뜻으로, 형제 요소 개수와 화면 너비에 관계없이 카드 너비를 항상 동일하게 맞춰줍니다. 실무에서 카드/그리드 레이아웃에 자주 쓰입니다.

4

[CSS 2] 박스 모델 (box-sizing)

버튼에 padding: 12px 20px; 을 줬더니 의도한 width: 200px보다 실제 렌더링 크기가 커져서 레이아웃이 깨졌습니다. 실무에서 가장 표준적으로 쓰는 해결 방법은?

.btn {
  width: 200px;
  padding: 12px 20px;
  border: 2px solid #000;
}

A) width를 200px보다 작게 여러 번 조정해서 맞춘다
B) * { box-sizing: border-box; } 를 리셋 규칙으로 추가해서 padding/border를 width 안에 포함시킨다
C) padding을 margin으로 바꾼다
D) border를 삭제한다

정답: B
box-sizing: border-box는 padding과 border를 width/height 안에 포함해서 계산하는 방식으로, 실무 CSS 리셋(reset)에 거의 항상 포함되는 규칙입니다.

5

[혼합] 반응형 시맨틱 레이아웃

아래처럼 HTML은 시맨틱하게 잘 짜여 있습니다.

<header>
  <h1>로고</h1>
  <nav class="menu">홈 | 소개 | 연락처</nav>
</header>

PC에서는 메뉴가 가로로 나열되길 원하지만, 모바일(화면 너비 600px 이하)에서는 세로로 쌓이길 원합니다. HTML 구조는 그대로 두고 CSS만으로 이를 구현하려면 어떻게 해야 할까요?

A) HTML에 모바일용 <nav> 태그를 하나 더 만들어 둔다
B) .menu { display: flex; } 로 기본 가로 배치를 만들고, @media (max-width: 600px) { .menu { flex-direction: column; } }로 모바일에서만 방향을 바꾼다
C) JavaScript로 화면 크기를 감지해서 태그를 통째로 교체한다
D) 처음부터 <nav>를 만들지 않고 <div>만 사용한다

정답: B
미디어 쿼리(@media)로 특정 화면 너비에서만 CSS 규칙을 덮어쓰는 것이 반응형 웹의 기본 원리입니다. HTML 구조는 하나만 유지하고, CSS만 화면 크기에 따라 다르게 적용하는 것이 실무 표준 방식입니다.

2. 전체 뼈대 — div/시맨틱 태그와 비교

header/nav/main/section/article/aside/footer를 "이름만 봐도 어떤 구역인지 아는 상자"라고 배웠습니다. 실제 페이지의 태그별 개수:

태그개수비고
<header>0없음
<nav>3상단 내비게이션 1개 + 내부 메뉴 컴포넌트 2개
<main>1페이지 전체를 감싸는 최상위 컨테이너
<section>1토스트 알림 영역(aria-live="polite")이지 본문 섹션 아님
<article>0없음
<aside>0없음
<footer>0없음 (하단 링크 영역은 그냥 div)
<h1>1"Open-source search infrastructure for AI"
<h2>/<h3>3 / 4소제목 계층
<body class="... bg-stone-50 text-foreground antialiased overflow-x-hidden">
  <main class="marketing-root relative overflow-x-hidden w-full min-h-screen grid grid-rows-[auto_1fr]">
    <div class="mb-62 z-10 flex-auto grid grid-cols-[1fr_min(80rem,100%-1rem)_1fr] ...">
      <nav class="col-2 w-full border-b lg:border-0"> ... </nav>
      <!-- 본문 내용도 전부 div -->
    </div>
  </main>
</body>

하단 링크 영역(흔히 "footer"라 부를 부분)은 진짜로는:

<h3 class="font-semibold text-sm mb-2">Product</h3>
<a href="/products/chromadb">Database</a>
...
<h3 class="font-semibold text-sm mb-2">Legal</h3>

피드백: "의미 있는 태그를 쓰자"는 원칙과 다르게, header/footer/article/aside를 아예 안 씁니다. Next.js + 컴포넌트 라이브러리(Base UI/Radix 계열) 기반 실무 프로젝트에서 자주 보이는 트레이드오프입니다. footer가 없으면 스크린리더가 contentinfo 랜드마크로 자동 인식 못 하므로 접근성 관점에선 아쉬운 지점. 반면 h1→h2→h3 제목 계층은 순서를 잘 지켰습니다.

3. 레이아웃 — Grid를 한 단계 더 쓴다

우리는 "똑같은 너비로 n등분"하는 grid-template-columns: repeat(2, 1fr)만 다뤘습니다. 이 사이트는:

grid-template-rows: auto 1fr;
grid-template-columns: 1fr min(80rem, 100% - 1rem) 1fr;

"1fr [본문] 1fr" 구조로 배경은 화면 끝까지 채우고, 내용물은 가운데 최대 80rem(1280px)로 제한합니다. min() 함수로 트랙 크기 자체를 계산하는 게 다음 단계로 배우면 좋은 실전 패턴입니다.

4. CSS 작성 방식 — Tailwind 유틸리티 클래스

외부/내부/인라인 스타일과 "선택자 하나에 속성 여러 개" 방식을 배웠는데, 이 사이트는 클래스 이름 자체가 CSS 속성인 Tailwind 방식입니다.

<div class="hidden lg:flex lg:items-center gap-6 lg:gap-2 ml-2 grow">

hidden→lg:flex→lg:items-center→gap-6 처럼 하나의 클래스 = 하나의 CSS 선언. 결과 CSS는 우리가 배운 것과 같지만, 클래스 이름을 매번 새로 짓지 않고 재사용할 수 있어서 실무에서 많이 씁니다.

5. CSS 변수 — 실제로는 "디자인 시스템"

:root { --main-color: red; } + var() 개념이, 실제로는 대규모 색상 팔레트로 쓰입니다.

:root {
  --background: #fff;
  --foreground: #0a0a0a;
  --primary: #171717;
  --secondary: #f5f5f5;
  --border: #e5e5e5;
  --ring: #a1a1a1;
  --radius: .625rem;
  --chart-1: #f05100;
  /* ... */
}

색의 의미(역할)로 이름 붙인 "디자인 토큰" 방식. --background 하나만 바꾸면 사이트 전체 색이 따라 바뀝니다. --radius 하나로 버튼·카드·인풋의 둥근 모서리를 통일하는 것도 특징.

:root { --background: lab(100% 0 0); --foreground: lab(2.75381% 0 0); ... }

같은 변수를 lab() 색공간으로 한 번 더 정의 — 와이드 갬멋 디스플레이 지원을 위한 점진적 향상(progressive enhancement)입니다.

6. 반응형 — max-width보다 진화한 방식

@media (max-width: 600px) { .box { flex-direction: column; } }

이 사이트가 실제로 쓰는 브레이크포인트:

@media (min-width: 40rem)  /* sm */
@media (min-width: 48rem)  /* md */
@media (min-width: 64rem)  /* lg */
@media (min-width: 80rem)  /* xl */
@media (min-width: 96rem)  /* 2xl */

차이점: (1) px 대신 rem을 써서 브라우저 글자 크기 배율에 반응형 기준점도 같이 늘어남(접근성에 유리). (2) max-width가 아니라 min-width 기준 — 모바일 퍼스트 설계.

@container field-group (min-width: 28rem) { ... }
@media (hover: hover) { ... }
@media (forced-colors: active) { ... }

@container: 화면이 아니라 부모 상자 크기 기준으로 스타일을 바꾸는 컨테이너 쿼리. (hover:hover): 마우스 기기에서만 호버 효과를 줘서 터치 기기 오작동을 방지. (forced-colors:active): 고대비 모드에서도 UI가 안 깨지게 하는 접근성 대응.

7. 애니메이션 — 제품을 설명하는 모션

@keyframes float + hover 조합처럼 커스텀 키프레임을 씁니다.

@keyframes flow-data { ... }
@keyframes flow-down { ... }
@keyframes reasoning-pulse { ... }
@keyframes blink-enterprise { ... }
@keyframes grow-line { ... }

"데이터가 흐른다", "AI가 추론하며 맥박친다", "선이 자라난다" — 제품 정체성을 시각적 은유로 표현하는 이름들. 순수 장식이 아니라 "이 회사가 뭘 하는지"를 모션으로 설명합니다.

8. 눈에 띄는 접근성 디테일

- 토스트 영역을 <section aria-live="polite">로 표시 — 새 알림이 뜨면 스크린리더가 자동으로 읽어줌
- 모달 제목에 화면엔 안 보이지만 스크린리더는 읽는 숨은 제목이 존재
- dark:aria-invalid 등 다크모드용 클래스가 html엔 다크모드가 없는데도 흔적으로 남아있음 → 공용 UI 컴포넌트 라이브러리를 그대로 가져다 쓴 흔적

감상평

감상평

Chroma는 사실 예쁘게 보이려는 사이트가 아니라 개발자한테 "이 도구 믿고 써도 된다"고 설득하려는 사이트다. 그래서 화려한 그래픽보다는 이 회사가 뭐 하는 곳인지, 얼마나 안정적인지 설명하는 데 힘을 준 티가 난다. 근데 그렇게 기능 설명이 목적인 사이트인데 정작 제일 기억에 남는 건 디자인이었다.

화면 자체는 그냥 흑백이다. 근데 이게 차갑거나 심심하지가 않다. 배경도 새하얀 흰색이 아니라 살짝 데운 미색이고 글자도 새까만 검정이 아니라 톤을 좀 낮춘 색이라, 모니터보다는 종이에 인쇄된 걸 보는 느낌에 가깝다. 굳이 꾸미지 않고 이렇게 절제된 흑백을 쓴 게 오히려 겉치레없이 솔직하게 다가와서 더 매력있게 느껴졌다.

제일 인상적이었던 건 곳곳에 있는 도트 패턴이다. 이미지가 아니라 옛날 컴퓨터 화면에서 보던 것 같은 네모 블록이랑 선 문자를 글자로 그대로 이어붙여서 다이어그램을 만들어놨고, 심지어 깜빡이면서 흐르는 애니메이션까지 붙어있다. 데이터가 흐른다는 개념을 화려한 그림 대신 옛날 서버실 터미널 느낌으로 표현해놓은 거라 보고 있으면 옛날 도트 프린터 영수증 같은 아날로그 느낌이 확 든다.

그림자도 요즘 사이트들처럼 뿌옇게 번지는 부드러운 그림자가 아니라 그냥 오른쪽 아래로 딱 밀어놓은 딱딱한 그림자를 쓴다. 종이를 오려서 붙인 스티커 같은 느낌이다. 폰트도 타자기 느낌 나는 고정폭 글씨체를 곳곳에 섞어 써서 전체적으로 오래된 인쇄물이나 터미널 느낌이 은근히 깔려있다.

이 사이트 header나 footer 같은 시맨틱 태그는 거의 안 쓰고 그냥 div에 클래스 이름표만 잔뜩 붙여서 기능 구현한 실용적인 코드다. 근데 방문자 입장에선 그런 뒷단은 전혀 안 보인다.

지금 이 미니 프로젝트 페이지 톤도 크로마를 보고 나서 흑백으로 다시 맞췄다. 카드 제목을 검정 바탕에 흰 글씨로 쓴 것도, 점선 구분선을 쓴 것도 이 사이트의 디자인을 참고했다.

기능 설명이 메인인 사이트인데 제일 기억에 남는 건 그 꾸밈없는 흑백 디자인이었다.
