# [openapi 15] MCP 개념과 공공 API·YOLO 실시간 분석 활용

> 작성: 2026-07-22

## Q&A

**Q. MCP(Model Context Protocol)가 뭔가?**

AI에게 새 능력을 추가해주는 "USB 같은 것"이다. LLM 혼자서는 인터넷 실시간 검색, 사내 DB 조회, 특정 API 호출 같은 걸 직접 못 하는데, MCP로 외부 도구를 연결해주면 AI가 그 도구를 불러써서 원래 못 하던 작업까지 수행할 수 있게 된다. 즉 LLM 자체의 성능을 올리는 게 아니라, LLM이 쓸 수 있는 "손발"의 종류를 늘려주는 규격이라고 보면 된다.

**Q. MCP 같은 도구 연결이 왜 필요한가? AI가 원래 다 알고 있는 거 아닌가?**

AI(LLM)는 학습 시점까지의 데이터로 지식을 갖고 있을 뿐, 실시간 정보나 학습 데이터에 없던 정보는 모른다. 문제는 모른다는 사실을 인정하기보다 그럴듯하게 지어내는 할루시네이션(hallucination) 현상이 있다는 점이다. 그래서 실시간 공공데이터(CCTV, 대중교통, 날씨 등)처럼 최신·정확한 값이 필요한 작업은 AI 혼자 답하게 두면 안 되고, 공식 API를 통해 실제 데이터를 가져와 AI에게 근거로 제공해야 신뢰할 수 있는 답이 나온다. MCP/공공 API 연동은 이 "근거 있는 답변"을 만들기 위한 장치다.

**Q. 공공 CCTV API + YOLO를 엮으면 실제로 뭘 할 수 있나?**

공공 데이터 포털에서 제공하는 실시간 CCTV 스트림 주소를 API로 받아온 뒤, 그 영상 프레임을 YOLO(컴퓨터 비전 탐지 모델)에 넣어 실시간으로 분석하는 구조다. 예를 들어 특정 도로 구간의 CCTV를 가져와 차량 수를 세거나 정체 여부를 판단하는 식으로, "공공 API로 데이터 소스 확보 + YOLO로 그 데이터를 실시간 분석"을 결합하는 응용 사례다.

## 개념 요약

| 개념 | 설명 |
|---|---|
| MCP (Model Context Protocol) | AI에게 외부 도구(API, DB, 파일 등) 접근 능력을 표준화된 방식으로 연결해주는 프로토콜 |
| 할루시네이션(Hallucination) | LLM이 모르는 내용을 그럴듯하게 지어내서 답하는 현상 |
| 공공 API | 정부/공공기관이 제공하는 실시간·정형 데이터 접근 창구 (CCTV, 교통, 기상 등) |
| AI + 공공API 결합 구조 | API로 실시간 사실 데이터 확보 → 그 데이터를 근거로 AI가 분석/판단 → 할루시네이션 방지 |
| YOLO 연계 | 공공 CCTV 스트림처럼 "영상 데이터를 제공하는 API"는 YOLO 같은 비전 모델의 실시간 입력 소스로 그대로 연결 가능 |

**실무 예시**: 스마트시티/교통관제 서비스에서 "실시간 교통량 조회 챗봇"을 만든다면, LLM에게 그냥 "지금 강남대로 정체 상황 알려줘"라고 물어보면 안 되고, 공공 CCTV API로 실제 영상을 가져와 YOLO로 차량 수를 세고, 그 숫자를 LLM에게 근거로 넘겨 자연어 답변만 생성시키는 구조가 정석이다. 이렇게 "사실 확인은 API/모델이, 설명은 LLM이" 역할을 나누는 게 MCP식 도구 연결의 핵심 활용법이다.
