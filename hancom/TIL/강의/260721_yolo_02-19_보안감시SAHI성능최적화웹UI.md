## 폴더별 코드로 보는 학습 흐름

`13_yolo/advanced` 폴더에 추가된 스크립트들을 순서대로 읽었을 때 드러나는 흐름 정리. 어제 다룬 `yolo_track.py`(추적) 이후 이어지는 실전 응용편이다.

**`solutions.*` 객체 공통 패턴**: `yolo_heatmap.py`, `yolo_alarm.py`, `yolo_blur.py`, `yolo_distance.py`, `yolo_region.py`, `yolo_in_out.py`, `yolo_speed.py`는 모두 `model = YOLO(...)` + `model(frame)` 대신 `solutions.Heatmap(...)`, `solutions.SecurityAlarm(...)` 같은 **전용 클래스 인스턴스**를 만들고 `인스턴스(frame)`으로 호출한다. 반환값도 `results[0].plot()`이 아니라 `results.plot_im`(이미 그려진 이미지 배열)이라는 점이 기존 `YOLO()` 사용법과 다르다.

**보안 감시 응용 (`yolo_alarm.py`, `yolo_blur.py`)**: `SecurityAlarm`은 탐지→추적→누적 카운트→임계치 도달 시 Gmail SMTP 메일 발송까지 내부에서 자동 처리한다. `ObjectBlurrer`는 탐지 박스 영역을 자동으로 모자이크 처리한다. 두 기능 모두 "탐지 결과를 어떻게 활용할 것인가"의 응용 사례.

**거리/영역 기반 로직 (`yolo_distance.py` → `yolo_distance_t.py`, `yolo_region.py`, `yolo_get_region.py`, `yolo_in_out.py`)**: 이 다섯 개는 "물체의 위치 관계"를 다루는 계열이다. `yolo_distance.py`는 `show=True`로 내장 시각화만 쓰고, `yolo_distance_t.py`는 `.process(frame)`으로 `pixels_distance` 값을 직접 뽑아 SAFE/Warning/DANGER 상태 판정 로직을 추가한 응용형이다. `yolo_get_region.py`(마우스 4클릭으로 좌표 추출) → `yolo_region.py`(그 좌표로 다각형 영역 내 객체 카운팅)는 이어지는 관계고, `yolo_in_out.py`는 같은 `ObjectCounter` 클래스에 점 2개(선)만 주면 IN/OUT 방향 카운팅으로 동작이 바뀐다.

**SAHI 소형 객체 탐지 (`yolo_sahi.py` → `yolo_sahi_org.py` → `yolo_sahi_slice.py`)**: 3단계로 나뉜 실습. 데모 이미지 다운로드 → 기본 추론(비교 기준 숫자 확보) → 슬라이싱 추론(타일로 쪼개서 탐지) 순서.

**속도 추정 (`yolo_speed.py`)**: `SpeedEstimator`에 `meter_per_pixel` 보정값을 직접 넣어야 실제 속도(km/h)가 나온다.

**세그멘테이션 응용 (`yolo_fast_sam.py`)**: 텍스트 프롬프트(`texts="dog"`)로 원하는 물체만 골라 분할하는 FastSAM.

**성능 최적화/배포 (`yolo_export.py` → `yolo_compare.py`)**: `model.export(format="openvino")`로 만든 `yolo11n_openvino_model/`을 `yolo_compare.py`에서 원본 `.pt`와 한 줄만 바꿔가며 FPS를 비교한다.

**웹 UI 배포 (`yolo_search.py`, `yolo_streamlit.py`)**: `solutions.SearchApp`(이미지 검색, `python`으로 직접 실행)과 `solutions.Inference`(실시간 대시보드, `streamlit run`으로 실행) — 둘 다 최상단에 SSL 인증서 검증 우회 코드가 붙어있다.

## Q&A

**Q. `solutions.Heatmap`, `SecurityAlarm` 같은 클래스들은 왜 `results[0].plot()`이 아니라 `results.plot_im`으로 결과 이미지를 꺼내는가?**

`YOLO("yolo11n.pt")`로 만든 기본 모델은 `model(frame)` 호출 시 탐지 결과 리스트(`results[0]`, `results[1]`...)를 반환해서 `.plot()`으로 그려야 하지만, `solutions.*` 클래스들은 애초에 "탐지 + 그리기 + (알람/블러/카운팅 등) 후처리"까지 한 번에 끝내는 고수준 래퍼(wrapper)라서 `SolutionResults`라는 별도 객체 하나를 반환한다. 이미 다 그려진 이미지가 `results.plot_im` 속성에 들어있어서 바로 `cv2.imshow()`에 넣을 수 있다. 기본 `YOLO()`와 `solutions.*`를 섞어 쓸 때 이 반환 타입 차이를 헷갈리면 `AttributeError`가 난다.

**Q. `yolo_distance.py`와 `yolo_distance_t.py`는 둘 다 `DistanceCalculation`을 쓰는데 왜 파일을 나눴나?**

`yolo_distance.py`는 `distance(frame)`을 호출하고 `show=True`가 내장 시각화 창까지 그려주는 "기본 동작 확인"용이다. `yolo_distance_t.py`는 같은 객체를 `distance.process(frame)`으로 호출해 반환값에서 `results.pixels_distance`(현재 선택된 두 객체 중심점 사이 픽셀 거리)를 직접 꺼내고, 이 숫자를 150px/100px 기준으로 SAFE·Warning·DANGER 상태로 분류하는 커스텀 로직을 얹은 응용형이다. `.process()`가 `__call__()`보다 로우레벨 접근을 제공해서, 시각화만으론 부족하고 "숫자 자체"가 필요할 때 쓴다.

**Q. `ObjectCounter`에 좌표 2개를 주면 IN/OUT 카운팅, `RegionCounter`엔 4개(다각형)를 주는데 이유가 뭔가?**

`yolo_in_out.py`의 `region=count_points`는 좌표 2개짜리 리스트라 "가상의 선"으로 해석되고, 그 선을 넘는 방향(왼→오, 오→왼 등)에 따라 IN/OUT을 판정한다. 반면 `yolo_region.py`의 `region=region_points`는 좌표 4개짜리 dict(`{"region-01": [...]}`)로 넘겨서, 그 4점이 이루는 **닫힌 다각형 내부**에 몇 개가 있는지를 센다. 같은 계열의 API라도 좌표 개수로 "선(경계 통과 카운트)"과 "면(영역 내부 카운트)"이 구분된다는 게 핵심.

**Q. SAHI 실습에서 `yolo_sahi_org.py`는 `demo_data/image_2.png`, `yolo_sahi_slice.py`는 `demo_data/image_1.png`를 쓰는데 이래도 되나?**

원래 취지(STEP 11 vs STEP 12 비교)는 "같은 이미지를 기본 추론 vs 슬라이싱 추론으로 각각 돌려서 탐지 개수 차이를 보는 것"이라, 서로 다른 이미지 파일을 쓰면 두 탐지 수 비교가 의미가 없어진다. `13_yolo/advanced/yolo_sahi_org.py:9`와 `yolo_sahi_slice.py:16`에 각각 다른 파일명이 들어있는 상태라 — 정확한 비교가 목적이라면 두 스크립트가 같은 이미지 경로를 가리키는지 확인이 필요하다.

**Q. `yolo_fast_sam.py`, `yolo_search.py`, `yolo_streamlit.py` 맨 위에 있는 `ssl._create_default_https_context = ssl._create_unverified_context`, `os.environ["CURL_CA_BUNDLE"] = ""` 코드는 뭘 우회하는 건가?**

FastSAM·SearchApp·Inference는 실행 시점에 내부적으로 모델 가중치나 임베딩 관련 파일을 인터넷에서 자동 다운로드하는데, 사내망/방화벽 환경에서는 SSL 인증서 체인이 신뢰되지 않아 `SSLCertVerificationError`가 나는 경우가 있다. 이 코드는 파이썬 전역 HTTPS 컨텍스트를 "인증서 검증 안 함"으로 바꾸고, `requests`/`curl` 계열 라이브러리가 참조하는 CA 번들 경로를 빈 값으로 지워서 검증 자체를 건너뛰게 만든다. 실무에서는 보안상 임시 우회로만 쓰고, 가능하면 사내 루트 인증서를 시스템에 등록하는 정공법을 쓰는 게 맞다.

## 개념 요약

| 개념 | 설명 |
|---|---|
| `solutions.*` 반환값 | `results.plot_im`으로 이미 그려진 이미지를 바로 받음 (기본 `YOLO()`의 `results[0].plot()`과 다름) |
| `SecurityAlarm` | 탐지 → 추적 → N회 누적 시 SMTP 메일 발송까지 자동 처리 |
| `ObjectBlurrer` | 탐지 박스 영역을 `blur_ratio`(0.0~1.0) 강도로 자동 모자이크 |
| `DistanceCalculation` | 마우스 좌클릭 2회로 고른 두 객체의 중심점 픽셀 거리 계산 |
| `RegionCounter` | 다각형(4점 이상) 영역 **내부** 객체 수 집계 |
| `ObjectCounter` | 2점(선)이면 IN/OUT 방향 카운팅으로 동작 |
| SAHI (Sliced Aided Hyper Inference) | 이미지를 작은 타일로 쪼개 각각 추론 후 합쳐서 작은 물체 탐지율을 높이는 기법 |
| `SpeedEstimator` | `meter_per_pixel` 보정값 기반으로 픽셀 이동량을 실제 속도(km/h)로 환산 |
| FastSAM 텍스트 프롬프트 | `texts="dog"`처럼 원하는 물체만 지정해 분할(SAM 전체 분할보다 훨씬 빠름) |
| OpenVINO export | `model.export(format="openvino")`로 Intel CPU 추론 최적화 모델 생성 |
| `time.perf_counter()` | 초 단위 오차 없는 경과 시간 측정 (Windows `time.time()`은 15.6ms 눈금이라 부정확) |

**1. 보안 감시 — 알람/블러**

```python
# yolo_alarm.py
google_alarm = solutions.SecurityAlarm(
    model="yolo11n.pt",
    records=2,      # 동일 객체 N회 감지 시 1회 메일
    classes=[2]     # 2 = 자동차만 탐지
)
google_alarm.authenticate(from_email, password, to_email)  # 1회만 실행 → 세션 유지
results = google_alarm(frame)   # 내부에서 탐지→추적→메일발송 자동 처리

# yolo_blur.py
blurrer = solutions.ObjectBlurrer(model="yolo11n.pt", blur_ratio=0.5)
results = blurrer(frame)
cv2.imshow("BLUR", results.plot_im)
```

**실무 예시**: `SecurityAlarm`은 무인 매장·주차장에서 지정 시간 이상 머무는 차량/사람을 자동 알림으로 연결할 때, `ObjectBlurrer`는 CCTV 영상을 외부에 공유하기 전 얼굴·번호판을 자동 익명화할 때 그대로 쓰는 패턴이다.

**2. 거리/영역 기반 로직**

```python
# yolo_distance_t.py — 거리 기반 상태 판정
results = distance.process(frame)
pixel_distance = results.pixels_distance

if pixel_distance >= 150:
    status = "SAFE"
elif pixel_distance >= 100:
    status = "Warning"
else:
    status = "DANGER"

# yolo_region.py — 다각형 영역 내부 카운팅
region_points = {"region-01": [(192, 175), (180, 410), (439, 386), (273, 168)]}
yolo_region = solutions.RegionCounter(model="yolo11n.pt", region=region_points, conf=0.4)
re_frame = cv2.resize(frame, (640, 480))   # 좌표 추출 시 해상도와 반드시 일치
results = yolo_region(re_frame)

# yolo_in_out.py — 2점(선) 기반 IN/OUT
count_points = [(234, 403), (579, 372)]
counter = solutions.ObjectCounter(model="yolo11n.pt", region=count_points)
```

**실무 예시**: `DistanceCalculation` + 상태 판정은 공사 현장에서 "작업자-중장비 간 안전거리 미달" 알림에, `RegionCounter`는 매장 특정 코너(예: 시식대) 체류 인원 집계에, `ObjectCounter`의 선 카운팅은 매장 입구 출입 인원수 실시간 집계(방문자 수 통계)에 그대로 쓰인다. `yolo_get_region.py`의 마우스 좌표 추출 스크립트는 이런 영역/선을 새 카메라 화각에 맞게 매번 다시 잡을 때 실무에서 좌표를 빠르게 뽑아내는 보조 도구로 쓰인다.

**3. SAHI — 작은 물체 탐지**

```python
# yolo_sahi_slice.py
detection_model = AutoDetectionModel.from_pretrained(
    model_type="ultralytics",
    model_path="yolo11n.pt",
    confidence_threshold=0.4
)
results = get_sliced_prediction(
    "demo_data/image_1.png",
    detection_model,
    slice_height=200, slice_width=200,
    overlap_height_ratio=0.1, overlap_width_ratio=0.1
)
results.export_visuals(export_dir="sahi/")
print(f"탐지 수: {len(results.object_prediction_list)}")
```

**실무 예시**: 드론/항공 촬영처럼 화면 속 물체가 몇 픽셀 안 되게 작게 찍히는 경우, 기본 추론은 원본 해상도로 한 번에 모델에 넣어서 작은 물체를 놓치기 쉬운데, SAHI는 이미지를 200×200 같은 작은 타일로 쪼개 각 타일을 확대해서 보는 효과를 내 소형 물체 탐지율을 끌어올린다. 실제로 `yolo_sahi_org.py`(기본)와 `yolo_sahi_slice.py`(슬라이싱)의 탐지 개수를 비교해보면 효과를 수치로 확인할 수 있다(단, 위 Q&A처럼 같은 이미지로 비교해야 의미가 있다).

**4. 속도 추정 · 성능 최적화**

```python
# yolo_speed.py
yolo_speed = solutions.SpeedEstimator(
    model="yolo11n.pt",
    max_speed=120,          # km/h 상한
    meter_per_pixel=0.5,    # 환경별 직접 보정 필수
    classes=[2],            # 차량만
)

# yolo_export.py
model = YOLO("yolo11n.pt")
model.export(format="openvino")   # yolo11n_openvino_model/ 폴더 생성

# yolo_compare.py — export 결과와 FPS 비교
model = YOLO("yolo11n_openvino_model/")   # 이 줄만 바꿔가며 비교
start_time = time.perf_counter()
results = model(frame, verbose=False)
end_time = time.perf_counter()
fps = 1 / (end_time - start_time)
```

**meter_per_pixel 보정 방법**: 길이를 아는 기준물(차선, 횡단보도 폭 등)을 화면에서 픽셀로 재고, `실제거리(m) ÷ 픽셀거리(px)`로 계산한다. 예: 차선 3m가 화면에서 6px이면 `meter_per_pixel = 0.5`.

**실무 예시**: `SpeedEstimator`는 도로 CCTV의 과속 감지 시스템에 그대로 들어가는 로직이고, 보정값을 잘못 잡으면 실제 속도와 무관한 숫자가 나오므로 설치할 카메라 화각마다 매번 다시 재야 한다. OpenVINO 변환 + FPS 비교는 실제 서비스를 GPU 없는 CPU 서버(엣지 디바이스, 매장 내 미니PC 등)에 배포하기 전에 "이 변환으로 실시간 처리가 가능한 수준인지"를 먼저 확인하는 필수 단계다.

**5. FastSAM · 웹 UI 배포**

```python
# yolo_fast_sam.py
model = FastSAM("FastSAM-s.pt")
results = model(source, texts="dog")   # 텍스트로 원하는 물체만 분할
output_image = results[0].plot()

# yolo_search.py — 이미지 검색 웹앱 (python으로 직접 실행)
app = solutions.SearchApp(device="cpu")
app.run(debug=True)

# yolo_streamlit.py — 실시간 대시보드 (streamlit run으로 실행)
inf = solutions.Inference(model="yolo11n.pt")
inf.inference()
```

**실무 예시**: FastSAM의 텍스트 프롬프트 분할은 "사진 속 특정 물체만 오려내기"가 필요한 이미지 편집 툴(배경 제거, 특정 객체만 합성)에 SAM보다 훨씬 빠르게 적용할 수 있다. `SearchApp`/`Inference`처럼 실행 방식이 다른 두 웹 UI를 구분해서 아는 게 중요한데, 사내 데모/PoC 단계에서 빠르게 결과를 보여줘야 할 때 코드를 새로 짜지 않고 이 둘을 그대로 붙여 쓰는 경우가 실무에서 흔하다.
