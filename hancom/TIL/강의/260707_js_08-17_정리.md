## Q&A

**Q. `n++` 과 `++n` 차이**

- `n++` — 현재 값 사용 후 +1
- `++n` — +1 후 값 사용
- 이 코드처럼 증가 후 바로 출력만 할 때는 결과 동일
- `n += 1`, `n = n + 1` 과도 같음

**Q. `querySelector` vs `createElement` 헷갈림**

```js
const li = document.querySelector("li")   // ❌ 기존 DOM에서 li를 찾음 → 없으면 null
const li = document.createElement("li")   // ✅ 새 li 요소를 메모리에 생성
```

- `querySelector` — 이미 있는 요소를 **찾는** 것
- `createElement` — 새 요소를 **만드는** 것
- 만든 요소는 `appendChild` 또는 `append`로 붙여야 화면에 나타남

---

**Q. `appendChild` vs `append` 차이**

| | `appendChild` | `append` |
|---|---|---|
| 인자 | 요소만 | 요소 또는 문자열 |
| 반환값 | 추가한 요소 반환 | undefined |
| 여러 개 동시 | 불가 | 가능 |

- 단순히 붙이기만 하면 `append`로 충분
- 추가한 요소를 바로 변수에 담아 이어서 쓸 때만 `appendChild` 필요

---

**Q. `out.innerHTML = ""` 왜 필요한가**

- `appendChild`는 **추가**만 하기 때문에 버튼을 여러 번 누르면 누적됨
- 클릭할 때마다 먼저 비우고 새로 채워야 함
- 반드시 for문 **밖**, 루프 **전**에 위치해야 함

**Q. `innerHTML` vs `textContent`**

```js
el.innerHTML = "<b>굵게</b>"   // → 굵게 표시됨 (HTML 해석)
el.textContent = "<b>굵게</b>" // → <b>굵게</b> 그대로 출력 (태그 무시)
```

- `innerHTML` — HTML 태그까지 해석해서 렌더링
- `textContent` — 문자 그대로 출력
- `<br>` 줄바꿈이 필요하면 `innerHTML` 사용

**Q. `const` vs `let` 구분**

```js
const btn = document.querySelector("#btn")  // 안 바뀜 → const
const text = s.value                        // 읽고 나서 안 바뀜 → const

let n = 0    // n++ 로 계속 바뀜 → let
let i = 10   // i-- 로 계속 바뀜 → let
```

- 기본적으로 `const` 쓰고, 나중에 값이 바뀌어야 할 때만 `let`
- 1회성이면 `const`

**Q. `for` vs `while` 언제 쓰나**

```js
// 횟수가 정해진 경우 → for
for (let i = 1; i <= count; i++) { ... }

// 조건이 언제 끝날지 모를 때 → while
while (i > 0) { i-- }
```

- 대부분 서로 바꿔 쓸 수 있음
- `for` — 카운터 변수가 괄호 안에 모여 있어서 명확
- `while` — 끝 기준이 불확실하거나 카운트다운처럼 최고점에서 시작할 때 자연스러움
- `while` 안에서 변화값(`i--`) 빠뜨리면 무한루프 주의

**Q. `render()` 함수를 따로 만드는 이유**

```js
const render = () => {
    out.textContent = fruits.join(", ")
    info.textContent = `개수: ${fruits.length}`
}

// 버튼마다 render() 한 줄로 끝
document.querySelector("#add").addEventListener("click", () => {
    fruits.push(fruit.value)
    render()
})
document.querySelector("#del").addEventListener("click", () => {
    fruits.pop()
    render()
})
```

- 화면 업데이트 코드를 버튼마다 반복 작성하지 않기 위해
- 버튼이 많아질수록 효과 큼
- 페이지 로드 시 초기값 표시를 위해 `render()`를 한 번 즉시 실행

**Q. `localStorage`란**

- 브라우저에 데이터를 저장하는 공간
- 새로고침하거나 껐다 켜도 유지됨
- 서버 없이 브라우저 안에만 저장 → 다른 사람은 못 봄

```js
localStorage.setItem("name", "콩이")   // 저장
localStorage.getItem("name")           // 불러오기 → "콩이"
localStorage.removeItem("name")        // 삭제
```

**Q. 배열 메서드**

```js
fruits.push("망고")              // 끝에 추가
fruits.pop()                     // 끝에서 제거 (인자 무시)
fruits.filter(f => f !== "망고") // 조건에 맞는 것만 남긴 새 배열
fruits.join(", ")                // 배열을 문자열로 이어붙임
```

- `pop()`은 인자를 넣어도 무시하고 무조건 마지막 제거
- 특정 항목 삭제는 `filter` 사용

**Q. 클래스(class)**

```js
class Dog {
    constructor(name, age) {  // new로 만들 때 자동 호출
        this.name = name      // 객체에 값 저장 (함수 끝나도 유지)
        this.age = age
    }
    bark() {
        return `${this.name}: 멍멍!`
    }
}

const poppy = new Dog("뽀삐", 3)  // 객체 생성
poppy.bark()                       // "뽀삐: 멍멍!"
```

- `class` — 틀(설계도)
- `constructor` — 키워드라 이름 못 바꿈, 내용은 자유
- `this` — 지금 만들어지는 객체 자신을 가리킴
- `new` — 틀로 객체를 찍어내는 키워드
- `this.name = name` — 함수가 끝나도 객체에 저장된 값은 유지됨
- 속성명과 메서드명이 같으면 충돌 → 메서드명을 다르게 (`getAge` 등)

**Q. `async/await` + `fetch`**

```js
// fetch — 서버에 요청 보내는 함수
// async/await — 응답 올 때까지 기다리게 하는 키워드 (세트로 사용)

async () => {
    const res = await fetch("/users")        // 요청 보내고 기다림
    const users = await res.json()           // 응답을 JS 객체로 변환
}
```

- `await` 없으면 응답 안 기다리고 다음 줄로 넘어가버림
- `async`는 "이 함수 안에서 await 쓸거야" 선언
- `res.json()` — 서버 응답(ReadableStream)을 JS가 읽을 수 있는 객체로 변환

**Q. CRUD 요청 방식**

| 방식 | 역할 | body 필요 |
|---|---|---|
| GET | 가져오기 | ❌ |
| POST | 추가하기 | ✅ |
| PUT | 수정하기 | ✅ |
| DELETE | 삭제하기 | ❌ |

```js
// GET (기본값, method 생략 가능)
fetch("/users")

// POST
fetch("/users", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ name: "John" })
})

// PUT (특정 id 수정)
fetch("/users/1", {
    method: "PUT",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ name: "Jane" })
})

// DELETE
fetch("/users/1", { method: "DELETE" })
```

**Q. json-server란**

- `db.json` 파일을 읽어서 자동으로 CRUD API를 만들어주는 도구
- 서버 코드 없이 연습용 백엔드로 사용
- `npx json-server@0.17.4 db.json --static .` 으로 실행
- 실행하면 `http://localhost:3000` 에서 서버 동작
- 터미널 끄면 서버도 같이 꺼짐

---

**Q. Live Server(5500) + json-server(3000) 포트 충돌 문제**

- 두 서버가 다른 포트라 `fetch("/users")`가 Live Server쪽으로 요청됨
- 해결: `--static .` 옵션으로 json-server가 정적 파일도 서빙
- `http://localhost:3000`에서 열면 `/users`로 바로 사용 가능

## 개념 요약

| 개념 | 한 줄 요약 |
|---|---|
| `createElement` | 새 요소 생성 (화면엔 아직 없음) |
| `appendChild/append` | 생성한 요소를 DOM에 붙여서 화면에 표시 |
| `innerHTML = ""` | 기존 내용 비우기 (누적 방지) |
| `render()` | 화면 업데이트 로직을 함수로 묶어 재사용 |
| `localStorage` | 브라우저에 영구 저장 (서버 불필요) |
| `class/constructor/new/this` | 객체 틀 만들고 찍어내는 문법 |
| `fetch` | 서버에 요청 보내는 함수 |
| `async/await` | 비동기 응답 기다리게 하는 키워드 |
| `json-server` | db.json으로 가짜 API 서버 자동 생성 |
