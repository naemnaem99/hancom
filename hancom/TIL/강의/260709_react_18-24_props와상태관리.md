# [React 18-24번] Props, useState, JSX 문법 정리

> 작성: 2026-07-09

## Q&A

**Q. 화살표 함수에서 `const Profile = () => ({job = "개발자"}) {` 처럼 쓰면 왜 에러가 나나?**

화살표 함수는 매개변수를 화살표 앞에 써야 한다. `() =>` 뒤에 다시 매개변수 목록 `({job})`을 쓰면 문법 오류다. 올바른 형태는 `const Profile = ({job = "개발자"}) => { ... }`. 괄호 위치를 착각해서 반복적으로 틀리기 쉬운 부분.

**Q. JSX에서 `class="card"`를 쓰면 왜 스타일이 안 먹나?**

HTML의 `class` 속성은 JSX에서 `className`으로 써야 한다. `class`는 JS 예약어(클래스 선언)와 겹치기 때문에 React가 별도 이름을 쓴다.

**Q. `import './components/23/Badge/jsx'`처럼 경로를 썼는데 컴포넌트가 안 보인다.**

두 가지 실수가 겹친 경우였다. 하나는 `.jsx` 확장자를 `/jsx`로 슬래시를 잘못 쓴 것(단순 오타), 다른 하나는 `import Badge from '...'`처럼 이름을 지정해서 가져와야 하는데 `import '...'`(이름 없이, side-effect only import)로 쓰면 `Badge`라는 변수 자체가 안 만들어져서 `<Badge/>`가 undefined 컴포넌트가 된다.

**Q. 컴포넌트가 props를 두 개의 객체로 나눠 받으면 (`({job}, {naem})`) 왜 두 번째 게 무시되나?**

React는 컴포넌트 함수에 props 객체를 **하나만** 넘긴다. 두 번째 매개변수 자리에 뭘 적어도 호출부에서 넘겨주는 값이 없으므로 항상 `undefined`다. 여러 값을 받고 싶으면 하나의 객체 안에 다 넣어야 한다: `({job = "개발자", naem = "김해냄"})`.

**Q. `{online && <p>👽</p>}` 이건 무슨 뜻인가?**

`&&`를 이용한 조건부 렌더링. `online`이 truthy면 뒤의 JSX가 반환되고, `false`/`null`/`undefined`면 그 값 자체가 반환되는데 React는 이 값들을 렌더링하지 않고 무시한다. 결과적으로 "true일 때만 표시"가 된다. 단, `online`이 `0`이나 `""` 같은 falsy 숫자/문자열이면 그 값 자체가 화면에 찍혀버릴 수 있어 boolean으로만 쓰는 게 안전하다.

**Q. 버튼 하나로 온라인/오프라인을 실시간으로 토글하려면?**

props로 받은 초기값은 고정이라 컴포넌트 내부에서 바꿀 수 없다. `useState`로 내부 상태를 만들고, 그 상태의 setter를 버튼 `onClick`에 연결해야 한다.

```jsx
import { useState } from 'react'

const Avatar = ({name, online}) => {
    const [isOnline, setIsOnline] = useState(online) // props를 초기값으로만 사용

    return (
        <>
        <h3>{name}</h3>
        <button onClick={() => setIsOnline(!isOnline)}>
            {isOnline ? '온라인' : '오프라인'}
        </button>
        </>
    )
}
```

**Q. 버튼(또는 상태 표시)을 정사각형으로 만들되 이름 폰트 크기와 항상 같게 하려면?**

고정 px 대신 `em` 단위를 쓰면 부모의 `font-size`를 기준으로 자동 계산된다. 이름과 버튼을 같은 부모 밑에 두고 그 부모에 `font-size`를 지정하면, 이름(`font-size: 1em`)과 버튼(`width/height: 1em`)이 항상 같은 크기를 유지한다.

```css
.avatar-row {
    display: flex;
    align-items: center;
    gap: 8px;
    font-size: 24px; /* 기준값 */
}

.avatar-name { font-size: 1em; margin: 0; }

.toggle-btn {
    width: 1em;
    height: 1em;
    border: none;
    border-radius: 4px;
}
```

**Q. `.card`에 `position: absolute;`를 추가하면 무슨 일이 생기나?**

문서 흐름에서 빠져서 카드가 차지하던 공간이 사라지고 형제 요소가 그 자리를 채운다. `top`/`left` 등을 안 주면 원래 자리 근처에 보이긴 하지만, 부모 중에 `position: relative`가 걸린 요소가 없으면 기준이 `<body>`(뷰포트)가 돼버려 스크롤에 따라 이상하게 움직일 수 있다. 특정 위치에 고정하려면 `top`/`left` 값 지정 + 부모에 `position: relative`가 필요하다.

**Q. `App.jsx`가 `App.css`를 import 안 해도 스타일이 남아있던데, 왜?**

`main.jsx`가 `index.css`를 전역으로 import하고 있기 때문. CSS import는 컴포넌트 범위로 격리되지 않고(CSS Module이 아닌 이상) 로드되는 순간 문서 전체에 적용된다. `main.jsx`는 앱의 진입점(`index.html`의 `<script type="module" src="/src/main.jsx">`)이라 `App.jsx`가 뭘 하든 상관없이 `index.css`는 항상 살아있다.

**Q. TypeScript(TS)와 React는 같은 건가?**

다른 레이어다. TypeScript는 언어(타입이 붙은 JS)이고, React는 UI를 컴포넌트로 만드는 라이브러리. 서로 독립적이라 조합이 3가지 가능하다: React 없이 TS만(백엔드 등), TS 없이 React만(`.jsx`, 지금 이 프로젝트), 둘 다(`.tsx`, 실무 표준). `.tsx`를 쓰면 prop 이름 오타나 잘못된 값 전달 같은 실수를 컴파일 시점에 잡아준다.

**Q. Badge 컴포넌트(`type`에 따라 색이 바뀌는 라벨)는 실무에서 언제 쓰나?**

짧은 텍스트에 상태/카테고리를 색으로 구분해 보여줄 때. 상품의 "NEW"/"품절" 태그, 알림 개수, 주문 상태("배송중"/"완료"/"취소"), PR 상태("Open"/"Merged") 등이 전형적인 예. 상태값 → 색상 매핑을 컴포넌트 안에 캡슐화해두면 호출하는 쪽은 의미(`type="new"`)만 넘기면 된다. 다만 삼항 연산자(`type === "new" ? 'green' : 'crimson'`)는 종류가 3개 이상으로 늘어나면 객체 매핑 방식으로 바꿔야 한다.

## 개념 요약

### 화살표 함수 props 받기

| 잘못된 형태 | 올바른 형태 |
|---|---|
| `() => ({job}) {}` | `({job}) => {}` |
| `({a}, {b}) => {}` (두 번째 무시됨) | `({a, b}) => {}` |

### JSX vs HTML 속성 차이

| HTML | JSX |
|---|---|
| `class` | `className` |
| 소문자 컴포넌트 태그 (`<profile/>`) | 대문자로 시작해야 컴포넌트로 인식 (`<Profile/>`) |

### import 방식

```js
import Card from './Card.jsx'   // 이름 지정 → 컴포넌트로 사용 가능
import './Card.css'             // 이름 없음 → 부수효과(스타일 적용)만, 변수 안 생김
```

### 조건부 렌더링 (`&&`)

```jsx
{isOnline && <p>👽</p>}
```
truthy면 렌더링, falsy(`false`/`null`/`undefined`)면 무시. `0`, `""`처럼 falsy이지만 값 자체가 찍히는 예외 케이스 주의.

### useState로 실시간 토글

```jsx
const [state, setState] = useState(초기값)
<button onClick={() => setState(!state)}>...</button>
```
props는 초기값 전달용으로만 쓰고, 실제로 바뀌는 값은 컴포넌트 내부 state로 관리.

### em 단위로 크기 동기화

같은 부모의 `font-size`를 기준으로 자식들에 `1em`을 주면, 부모 값만 바꿔도 자식 크기가 같이 조정된다. 픽셀 하드코딩보다 유지보수에 유리.

### position: absolute

| 상태 | 동작 |
|---|---|
| 부모에 `position: relative` 없음 | `<body>`(뷰포트) 기준으로 배치, 문서 흐름에서 빠짐 |
| 부모에 `position: relative` 있음 | 그 부모 기준으로 `top`/`left` 배치 |

### CSS 적용 범위

`main.jsx`(진입점)에서 import한 `index.css`는 전역 적용. 개별 컴포넌트의 CSS import 여부와 무관하게 항상 살아있음.

### TypeScript vs React

| 조합 | 확장자 | 특징 |
|---|---|---|
| React only | `.jsx` | 빠른 학습/프로토타이핑 |
| React + TS | `.tsx` | 실무 표준, 타입으로 prop 오타 방지 |
| TS only | `.ts` | React 없이 백엔드/로직 코드 |

### Badge 패턴

상태값 → 색상 매핑을 컴포넌트 내부에 캡슐화. 호출부는 의미만 전달.

```jsx
const Badge = ({text, type}) => {
    const color = type === "new" ? 'green' : 'crimson'
    return <span style={{backgroundColor: color, color: '#fff'}}>{text}</span>
}
```
타입 종류가 늘어나면 삼항 대신 객체 매핑(`{new: 'green', sold: 'gray', sale: 'crimson'}[type]`)으로 전환 필요.
