# [tool 02] 터미널 경로 자동완성 & npm run dev의 원리

> 작성: 2026-07-09

## Q&A

**Q. 터미널에서 `cd` 뒤에 폴더 경로 칠 때 `\`를 매번 직접 안 치는 방법은?**

`Tab` 키로 자동완성한다. 폴더 이름 앞글자 몇 개만 치고 `Tab`을 누르면 나머지 경로(백슬래시 포함)가 채워진다.

```
cd Desk[Tab]              → cd Desktop\
cd Desktop\han[Tab]       → cd Desktop\hancom\
```

이름이 겹치는 후보가 여러 개면 `Tab`을 여러 번 눌러 순환한다.

**Q. `my-app` 말고 그 상위 폴더인 `05_react`에서 `npm run dev`를 실행하면 어떻게 되나?**

에러가 난다. `package.json`이 `05_react/my-app/`에만 있고 `05_react/`에는 없기 때문이다.

```
npm error code ENOENT
npm error enoent Could not read package.json
```

`npm`은 **현재 디렉터리**에서만 `package.json`을 찾고, 하위 폴더까지 뒤져서 자동으로 찾아주지 않는다. 그래서 `cd my-app`으로 들어간 다음에 실행해야 한다.

**Q. `npm run dev`는 서버를 여는 명령인데, 왜 `package.json`이 꼭 필요한가?**

`npm run dev`는 "서버를 켜라"는 고정 명령이 아니라, **`package.json`의 `scripts.dev`에 적힌 명령어를 대신 실행해달라**는 뜻이기 때문이다.

```json
"scripts": {
    "dev": "vite",
    "build": "vite build"
}
```

즉 `npm run dev` = `npx vite`를 실행하라는 것뿐이고, `dev`라는 이름 자체도 프로젝트마다 임의로 붙인 것이다(다른 프로젝트는 `start`, `serve` 등을 쓰기도 함). `package.json`이 없으면:
1. `dev`가 실제로 뭘 실행해야 하는지 npm이 알 방법이 없고
2. 프로젝트가 뭘 의존하는지(react, vite 등)도 알 수 없어서 `node_modules` 설치 여부도 판단 불가

## 개념 요약

| 상황 | 이유 |
|---|---|
| `cd` 경로 자동완성 | `Tab` 키가 백슬래시 포함 경로를 대신 채워줌 |
| 상위 폴더에서 `npm run dev` 실패 | npm은 현재 디렉터리에서만 `package.json`을 찾음 (하위 탐색 X) |
| `npm run dev`가 곧 서버 실행이 아님 | `package.json`의 `scripts.dev`에 적힌 명령을 대신 실행하는 것 |
| `package.json`이 필요한 이유 | 실행할 명령 정의 + 프로젝트 의존성(react, vite 등) 목록 역할 |

**핵심**: `npm run <이름>`은 그 자체로 의미를 갖는 명령이 아니라, `package.json.scripts.<이름>`을 대신 실행해주는 래퍼(wrapper)다. `package.json`이 없거나 잘못된 폴더에서 실행하면 npm은 무엇을 실행해야 할지 알 수 없다.
