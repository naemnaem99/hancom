# [React 40] react-router-dom으로 페이지 라우팅하기

> 작성: 2026-07-13

## Q&A

**Q. `npm install react-route-dom`을 했더니 404 에러가 난다. 왜 그런가?**

패키지 이름 오타다. 올바른 이름은 `router`가 들어간 `react-router-dom`이다. npm 레지스트리에는 정확한 패키지명으로만 검색되기 때문에 `route`(경로)와 `router`(경로를 관리하는 주체)를 헷갈리면 그대로 404가 난다.

```bash
npm install react-router-dom
```

**Q. "라우팅(routing)한다"는 게 정확히 무슨 뜻인가?**

**주소(URL)에 따라 다른 컴포넌트를 화면에 보여주는 것**을 말한다. 기존 멀티페이지 웹사이트는 `home.html`, `about.html`처럼 실제 파일이 여러 개 있고, 링크를 클릭하면 서버에 새 HTML을 요청해 페이지 전체가 새로고침된다. 반면 React SPA는 실제로는 `index.html` 파일 하나만 존재하고, `/`, `/about` 같은 경로는 서버 파일이 아니라 **브라우저 안에서만 관리되는 가짜 경로**다. `react-router-dom`이 "현재 주소가 무엇인지" 감시하다가 그에 맞는 컴포넌트로 화면 일부만 바꿔치기 해준다.

**Q. `<Route path="/about" element={<About/>}/>`을 썼는데 화면에 아무 일도 안 일어난다. `<link to="/about">`도 같이 썼다.**

소문자 `<link>`로 쓰면 HTML의 `<link>` 태그(스타일시트 등을 연결할 때 쓰는 태그)로 인식되어 `react-router-dom`의 `Link` 컴포넌트가 아니게 된다. React 컴포넌트는 관례상 대문자로 시작해야 JSX가 "이건 커스텀 컴포넌트다"라고 구분하므로, 반드시 대문자 `<Link to="/about">`로 써야 한다.

```jsx
import { BrowserRouter, Routes, Route, Link } from 'react-router-dom'

function App() {
  return (
    <BrowserRouter>
      <nav>
        <Link to="/">Home</Link>
        <Link to="/about">About</Link>
      </nav>
      <Routes>
        <Route path="/" element={<Home/>}/>
        <Route path="/about" element={<About/>}/>
      </Routes>
    </BrowserRouter>
  )
}
```

**Q. `index.html`을 VSCode Live Server("Go Live")로 직접 열면 왜 안 되나?**

이 프로젝트는 Vite의 변환(transform) 과정을 거쳐야만 실행되는 코드로 짜여 있어서다. 두 가지가 문제된다.

1. **JSX는 브라우저가 못 읽음**: `<App/>`, `<BrowserRouter>` 같은 JSX 문법은 `React.createElement(...)` 형태로 변환돼야 브라우저가 실행할 수 있다. Vite dev server는 요청이 올 때마다 이 변환을 즉석에서 해주지만, Live Server는 파일을 있는 그대로 던져주기만 한다.
2. **패키지 이름만 적힌 import를 못 찾음**: `import { createRoot } from 'react-dom/client'`처럼 경로가 아니라 패키지 이름만 적힌 import(bare specifier)는 브라우저가 그대로 해석할 수 없다. Vite는 이런 요청을 `node_modules` 안의 실제 파일로 매핑해주지만 Live Server에는 그 기능이 없어 404가 난다.

## 개념 요약

| 개념 | 설명 |
|---|---|
| SPA 라우팅 | URL 경로와 렌더링할 컴포넌트를 짝지어주고, 브라우저 히스토리(뒤로가기 등)까지 관리해주는 방식 |
| `BrowserRouter` | 앱 전체를 감싸서 라우팅 기능을 활성화하는 컨테이너 |
| `Routes` / `Route` | 경로별로 어떤 컴포넌트를 보여줄지 매핑하는 테이블 역할 |
| `Link` | `<a>`와 달리 페이지 전체를 새로고침하지 않고 주소만 바꿔 컴포넌트를 교체 |

**실무 예시**: 쇼핑몰 사이트에서 `/products`, `/products/:id`, `/cart`처럼 화면이 여러 개 필요할 때, 페이지마다 HTML 파일을 따로 만드는 대신 `react-router-dom`으로 경로별 컴포넌트를 등록해두면 새로고침 없이 빠르게 화면이 전환되고, 상세 페이지 같은 동적 경로(`:id`)도 하나의 컴포넌트로 재사용할 수 있다.

```jsx
// 폴더 구조 예시
// src/pages/Home.jsx   → "/" 에 매핑되는 페이지 단위 컴포넌트
// src/pages/About.jsx  → "/about" 에 매핑되는 페이지 단위 컴포넌트
// src/components/      → 여러 페이지에서 재사용되는 UI 조각
```

**실무 예시**: 프로젝트가 커지면 `components/`(재사용 UI 조각)와 `pages/`(라우팅되는 화면 단위)를 폴더로 분리해두는 게 관례다. 어떤 파일이 실제 주소와 연결된 "화면"인지, 어떤 파일이 여기저기서 갖다 쓰는 "부품"인지 한눈에 구분되기 때문이다.
