# [Web 01] SVG란 무엇인가

> 작성: 2026-07-03

## Q&A

**Q. 웹사이트의 아이콘이나 로고가 PNG 파일인지 CSS 구현인지 어떻게 구분하나?**

개발자 도구(F12)의 Elements 탭에서 해당 요소를 확인하면 된다.
- `<img src="...png">` → PNG 파일
- `<svg><path d="..."/></svg>` → 인라인 SVG (코드로 직접 그린 것)
- CSS `clip-path`, `polygon` → 순수 CSS 도형

**Q. SVG가 정확히 뭔가? PNG/JPG와 어떻게 다른가?**

SVG(Scalable Vector Graphics)는 **수학적 좌표로 그림을 그리는 코드**다.
PNG/JPG는 픽셀 하나하나의 색상 데이터를 저장하지만, SVG는 "이 좌표를 이렇게 연결해서 그려라"는 명령을 텍스트로 저장한다.

**Q. 실제 사이트에서 SVG 아이콘을 직접 가져올 수 있나?**

가능하다. 브라우저 개발자 도구에서 해당 `<svg>` 태그를 우클릭 → **Copy → Copy outerHTML** 하면 SVG 코드를 그대로 복사할 수 있다. 이걸 `.svg` 파일로 저장하면 재사용 가능하다.

**Q. Next.js 같은 프레임워크 사이트에서 아이콘은 어디에 있나?**

React 컴포넌트 안에 인라인 SVG로 삽입되어 있고, 번들된 `.js` 파일에 포함되어 있다. 별도 이미지 파일로 존재하지 않는다.

## 개념 요약

| | PNG / JPG | SVG |
|---|---|---|
| 저장 방식 | 픽셀 색상 데이터 | 수학 공식 (텍스트) |
| 확대 시 | 깨짐 | 절대 안 깨짐 |
| 파일 크기 | 크다 | 단순 도형은 매우 작다 |
| 사진 표현 | 완벽 | 불가능 |
| CSS로 색 변경 | 불가 | 가능 |
| 코드로 수정 | 불가 | 가능 (텍스트 파일이라서) |

```xml
<!-- SVG 코드 예시: 비행기(화살표) 모양 -->
<svg viewBox="0 0 24 24">
  <path d="M2 12L22 2L12 22L10 14L2 12Z" />
  <!-- d 속성의 좌표값들이 연결되어 도형을 만든다 -->
</svg>
```

**핵심**: SVG는 그냥 텍스트 파일이다. 메모장으로도 열 수 있고, 좌표만 바꾸면 모양이 바뀐다.
