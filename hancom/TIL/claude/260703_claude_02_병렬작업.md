# [Claude 02] 같은 세션에서 두 작업 동시에 처리하기

> 작성: 2026-07-03 | 수정: 2026-07-08

## Q&A

**Q. Claude Code 세션 하나에서 두 가지 작업을 동시에 실행할 수 있는가?**

가능하다. 크게 4가지 방법이 있다.

**Q. 독립적인 두 파일 읽기처럼 연관 없는 작업은 어떻게 병렬 처리되나?**

Claude가 자동으로 판단해서 병렬 툴 호출로 실행한다. 별도 지시 없이 동시에 처리된다.

**Q. 빌드나 테스트처럼 오래 걸리는 작업을 백그라운드로 넘기고 다른 작업을 하려면?**

Bash 툴의 `run_in_background: true` 옵션을 쓰면 된다. 완료되면 자동으로 알림이 온다.

**Q. 완전히 독립된 두 대화를 동시에 진행하고 싶다면?**

Claude Code는 단일 대화 세션이라 불가능하다. 터미널 탭을 두 개 열어 각각 `claude` 를 실행하는 것이 실용적인 방법이다.

---

## 개념 요약

| 방법 | 설명 | 특징 |
|---|---|---|
| 병렬 툴 호출 | 독립 작업을 Claude가 자동으로 동시 실행 | 별도 지시 불필요 |
| 백그라운드 서브에이전트 | `Agent` 툴에 `run_in_background: true` | 완료 시 자동 알림 |
| 백그라운드 Bash | Bash 툴에 `run_in_background: true` | 빌드·테스트 등 긴 명령에 유용 |
| 터미널 탭 분리 | 탭 2개에서 각각 `claude` 실행 | 완전히 독립된 세션 |

**핵심**: 하나의 세션 안에서는 백그라운드 실행으로 병렬 처리가 가능하지만, 두 개의 독립적인 대화가 필요하다면 세션을 분리해야 한다.

## [수정: 2026-07-08] 작업별 모델 선택 & 서브에이전트 모델 배정

**Q. 대화 도중에 Claude가 스스로 자기 모델을 바꿀 수 있나?**

불가능하다. 모델 선택은 세션 시작 시 사용자가 `/model` 명령으로 고정하는 설정이고, Claude 자신에게는 세션 도중 모델을 전환할 수 있는 도구가 없다. Opus에서 속도만 바꾸는 `/fast` 토글이 있지만, 이건 같은 모델을 더 빠르게 돌리는 것이지 모델 자체를 바꾸는 게 아니다.

**Q. 그럼 작업 유형에 따라 다른 모델을 쓰고 싶으면 어떻게 하나?**

두 가지 방법이 있다.
1. 세션을 시작하기 전에 작업 성격에 맞춰 `/model`로 직접 고른다 (예: 단순 반복 작업엔 Haiku, 일반 작업엔 Sonnet, 복잡한 리팩토링엔 Opus).
2. `Agent` 툴로 서브에이전트를 띄울 때 `model` 파라미터로 그 서브에이전트만 다른 모델을 지정한다. 메인 세션의 모델은 그대로 유지하면서, 하위 작업만 다른 모델에 위임하는 방식이다.

**Q. 서브에이전트에 모델을 나눠 맡기는 설계는 실제로 어떻게 하나?**

오케스트레이터(메인 세션)는 종합 판단이 필요한 부분만 맡고, 단순 데이터 수집/탐색은 저렴한 모델의 서브에이전트에 위임하는 구조가 기본이다.

```
오케스트레이터 (Sonnet) — 최종 종합·판단
├── 서브에이전트 A (Haiku) — 파일/로그 단순 조회
├── 서브에이전트 B (Haiku) — 다른 영역 단순 조회 (병렬 실행)
└── 종합: 오케스트레이터가 A, B 결과를 취합해 결론 도출
```

**Q. 서브에이전트를 무조건 쓰는 게 이득인가?**

아니다. 서브에이전트 기동에는 콜드스타트·컨텍스트 재구성 비용이 있다. 대상 범위가 작으면(파일 수십 개 수준) 직접 처리하는 것이 더 빠르고 저렴하다. 서브에이전트가 이득을 보는 기준은:
- 대상 범위가 커서 병렬화 이득이 명확할 때 (파일 수백 개, 커밋 수천 개 등)
- 각 서브태스크가 독립적이라 메인 컨텍스트를 오염시키지 않는 게 중요할 때
- 탐색 결과가 방대해서 메인 세션 컨텍스트 창을 아끼고 싶을 때

## 개념 요약 (모델 선택)

| 모델 | 적합한 작업 | 특징 |
|---|---|---|
| Haiku 4.5 | 반복적·기계적 작업 (스캐폴딩, 단순 조회) | 빠르고 저렴, 판단·추론 불필요할 때 |
| Sonnet 5 (기본) | 코딩 실습, 글쓰기, 일반 Q&A | 대부분의 작업에서 균형 잡힌 선택 |
| Opus 4.8 | 다중 파일 리팩토링, 복잡한 버그 추적, 고품질 장문 콘텐츠 | 추론·종합력이 중요할 때 |

**핵심**: 모델 실시간 자동 전환은 불가능하지만, 서브에이전트별로 다른 모델을 배정하는 것은 가능하다. 오케스트레이터는 판단이 필요한 부분만 담당하고, 단순 반복 작업은 저렴한 모델의 서브에이전트에 위임하는 것이 비용 효율적인 설계다.
