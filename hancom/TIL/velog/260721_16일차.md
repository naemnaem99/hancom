## 도구 & 환경

### SSL 인증서 검증 우회로 모델 다운로드 문제 해결

FastSAM, 이미지 검색 웹앱, Streamlit 대시보드처럼 실행 시점에 모델 가중치나 관련 파일을 인터넷에서 자동으로 내려받는 스크립트들은, 사내망이나 방화벽 환경에서 SSL 인증서 체인이 신뢰되지 않아 `SSLCertVerificationError`가 나는 경우가 있다. 그래서 스크립트 맨 위에 인증서 검증 자체를 건너뛰는 코드를 추가했다.

```python
import ssl, os
ssl._create_default_https_context = ssl._create_unverified_context
os.environ["CURL_CA_BUNDLE"] = ""
os.environ["REQUESTS_CA_BUNDLE"] = ""
```

파이썬 전역 HTTPS 컨텍스트를 "검증 안 함"으로 바꾸고, `requests`/`curl` 계열 라이브러리가 참조하는 CA 인증서 번들 경로를 빈 값으로 지워버리는 방식이다.

**실무 예시**: 이 방식은 로컬 개발/PoC 단계의 임시 우회로만 쓰는 게 맞다. 실제 운영 환경이나 보안이 중요한 사내망에서는 검증을 끄는 대신 사내 루트 인증서를 시스템 트러스트 스토어에 등록하는 정공법을 써야 한다.

## 코딩 학습

### solutions 클래스, 반환값이 다르다

`YOLO("yolo11n.pt")`로 만드는 기본 모델은 `model(frame)`을 호출하면 탐지 결과 리스트를 반환해서 `results[0].plot()`으로 직접 그려야 했다. 반면 오늘 다룬 `solutions.Heatmap`, `solutions.SecurityAlarm`, `solutions.ObjectBlurrer` 같은 클래스들은 "탐지 + 시각화 + 알람/블러/카운팅 같은 후처리"까지 한 번에 끝내는 고수준 래퍼라서, 이미 다 그려진 이미지가 `results.plot_im` 속성에 그대로 들어있다.

```python
heatmap = solutions.Heatmap(model="yolo11n.pt", colormap=cv2.COLORMAP_MAGMA)
results = heatmap(frame)
annotated_frame = results.plot_im   # results[0].plot()이 아니라 이렇게 꺼낸다
cv2.imshow("HEATMAP", annotated_frame)
```

**실무 예시**: 기본 `YOLO()`와 `solutions.*`를 같은 코드베이스에서 섞어 쓸 때 이 반환 타입 차이를 놓치면 `AttributeError`가 발생한다. "이 객체가 탐지만 하는 모델인지, 후처리까지 포함한 솔루션인지"를 먼저 확인하는 습관이 필요하다.

### 보안 감시: 알람과 블러

`SecurityAlarm`은 탐지 → 추적 → 누적 카운트 → 임계치 도달 시 Gmail SMTP 메일 발송까지 내부에서 자동으로 처리해준다.

```python
google_alarm = solutions.SecurityAlarm(
    model="yolo11n.pt",
    records=2,      # 동일 객체 N회 감지 시 1회 메일 (기본 5)
    classes=[2]     # 2 = 자동차만 탐지 대상
)
google_alarm.authenticate(from_email, password, to_email)  # 1회만 실행 → 세션 유지

results = google_alarm(frame)   # 탐지→추적→메일발송이 이 한 줄 안에서 다 처리됨
```

메일 발송에는 일반 로그인 비밀번호가 아니라 Gmail 앱 비밀번호(16자리)가 필요하다.

`ObjectBlurrer`는 탐지된 박스 영역을 자동으로 모자이크 처리한다.

```python
blurrer = solutions.ObjectBlurrer(model="yolo11n.pt", blur_ratio=0.5)  # 0.0~1.0
results = blurrer(frame)
```

**실무 예시**: `SecurityAlarm`은 무인 매장·주차장에서 특정 차량이나 사람이 일정 시간 이상 머무를 때 담당자에게 자동 알림을 보내는 로직으로 바로 활용할 수 있다. `ObjectBlurrer`는 CCTV 영상을 외부에 공유하기 전 얼굴이나 번호판을 자동으로 익명화할 때 쓰는 패턴이다.

| 항목 | 설명 |
|---|---|
| `records` | 동일 객체가 몇 번 감지되면 메일을 보낼지 |
| `classes` | 알람 대상으로 삼을 COCO 클래스 번호 |
| `blur_ratio` | 블러 강도, 0.0(약함)~1.0(강함) |

### 거리·영역 기반 판정 로직

`DistanceCalculation`은 화면에서 마우스 좌클릭 두 번으로 대상을 고르면 두 객체 중심점 사이의 픽셀 거리를 계산해준다. 처음엔 내장 시각화(`show=True`)만 확인했는데, 이 숫자를 직접 꺼내 상태를 판정하는 응용도 해봤다.

```python
results = distance.process(frame)          # __call__ 대신 로우레벨 접근
pixel_distance = results.pixels_distance

if pixel_distance >= 150:
    status = "SAFE"
elif pixel_distance >= 100:
    status = "Warning"
else:
    status = "DANGER"
```

영역/선 카운팅은 좌표 개수로 동작이 갈린다. 좌표 2개를 주면 "가상의 선"으로 해석되어 그 선을 넘는 방향에 따라 IN/OUT을 세고, 좌표 4개(다각형)를 주면 그 영역 **내부**에 있는 객체 수를 센다.

```python
# 선(2점) → 방향별 IN/OUT 카운팅
count_points = [(234, 403), (579, 372)]
counter = solutions.ObjectCounter(model="yolo11n.pt", region=count_points)

# 다각형(4점) → 영역 내부 카운팅
region_points = {"region-01": [(192, 175), (180, 410), (439, 386), (273, 168)]}
yolo_region = solutions.RegionCounter(model="yolo11n.pt", region=region_points, conf=0.4)

re_frame = cv2.resize(frame, (640, 480))   # 좌표를 딸 때와 추론할 때 해상도 반드시 일치
results = yolo_region(re_frame)
```

이런 좌표는 매번 손으로 계산하지 않고, 마우스 클릭으로 직접 추출한다.

```python
def mouse_callback(event, x, y, flags, param):
    if event == cv2.EVENT_LBUTTONDOWN:
        points.append((x, y))
        print(f"클릭된 좌표는 {x, y} 입니다.")

cv2.setMouseCallback("GET_X_Y", mouse_callback)
```

**실무 예시**: 거리 기반 상태 판정은 공사 현장에서 작업자와 중장비 사이 안전거리 미달을 감지하는 로직에, 다각형 영역 카운팅은 매장의 특정 코너(시식대 등) 체류 인원 집계에, 선 기반 IN/OUT 카운팅은 매장 입구 방문자 수 실시간 통계에 그대로 쓰인다. 카메라 화각이 바뀔 때마다 마우스로 좌표를 새로 따야 하므로, 좌표 추출용 스크립트를 따로 두는 게 실무에서도 일반적이다.

### SAHI로 작은 물체 탐지율 높이기

SAHI(Sliced Aided Hyper Inference)는 이미지를 작은 타일로 쪼개서 각 타일마다 따로 추론한 뒤 결과를 합치는 기법이다. 원본 해상도로 한 번에 추론하면 화면 속 물체가 몇 픽셀 안 되게 작을 때 놓치기 쉬운데, 타일로 쪼개 확대해서 보는 효과를 내면 탐지율이 올라간다.

```python
detection_model = AutoDetectionModel.from_pretrained(
    model_type="ultralytics",
    model_path="yolo11n.pt",
    confidence_threshold=0.4
)

results = get_sliced_prediction(
    "demo_data/image_1.png",
    detection_model,
    slice_height=200, slice_width=200,       # 타일 크기(px)
    overlap_height_ratio=0.1, overlap_width_ratio=0.1   # 타일 간 겹침 10%
)

print(f"탐지 수: {len(results.object_prediction_list)}")
```

기본 추론과 슬라이싱 추론의 탐지 개수를 나란히 비교해보면 SAHI 효과를 수치로 확인할 수 있다. 다만 정확히 비교하려면 두 스크립트가 같은 이미지를 가리켜야 하는데, 실습 중 사용한 두 파일의 이미지 경로가 서로 달라(`image_2.png` / `image_1.png`) 정확한 비교가 되려면 경로를 맞출 필요가 있었다.

**실무 예시**: 드론이나 항공 촬영처럼 물체가 화면에서 작게 찍히는 상황(교통 모니터링, 항만 컨테이너 탐지 등)에서 기본 추론 대신 SAHI를 적용하면 소형 객체 탐지 누락을 크게 줄일 수 있다.

### 속도 추정과 성능 최적화

`SpeedEstimator`는 픽셀 이동량을 실제 속도(km/h)로 환산해준다. 이때 핵심은 `meter_per_pixel` 보정값인데, 길이를 아는 기준물(차선, 횡단보도 폭 등)을 화면에서 픽셀로 잰 뒤 `실제거리(m) ÷ 픽셀거리(px)`로 계산한다. 예를 들어 차선 3m가 화면에서 6px로 찍혔다면 `meter_per_pixel = 0.5`가 된다.

```python
yolo_speed = solutions.SpeedEstimator(
    model="yolo11n.pt",
    max_speed=120,           # km/h 상한
    meter_per_pixel=0.5,     # 환경(카메라 설치 위치)마다 직접 보정 필수
    classes=[2],             # 차량만 추적
)
```

실시간 처리 속도(FPS)를 직접 재고 싶을 때는 `time.time()` 대신 `time.perf_counter()`를 써야 한다. Windows에서 `time.time()`은 15.6ms 단위로만 눈금이 찍혀서, 50ms짜리 추론 하나를 재면 오차가 ±30%까지 벌어질 수 있기 때문이다.

```python
start_time = time.perf_counter()
results = model(frame, verbose=False)
end_time = time.perf_counter()
fps = 1 / (end_time - start_time)
```

CPU에서 더 빠르게 돌리고 싶다면 모델을 OpenVINO 형식으로 변환한 뒤 원본과 FPS를 비교해본다.

```python
# 변환
model = YOLO("yolo11n.pt")
model.export(format="openvino")   # yolo11n_openvino_model/ 폴더 생성

# 비교 — 이 한 줄만 바꿔가며 실행
model = YOLO("yolo11n_openvino_model/")
```

**실무 예시**: `SpeedEstimator`는 도로 CCTV 과속 감지 시스템에, 보정값을 카메라 설치 위치마다 새로 재는 과정은 실제 현장 배포에서 빠질 수 없는 단계다. OpenVINO 변환과 FPS 비교는 GPU 없는 CPU 서버(매장 내 미니PC, 엣지 디바이스)에 서비스를 배포하기 전에 실시간 처리가 가능한 수준인지 미리 검증하는 필수 절차다.

### FastSAM과 웹 UI 배포

FastSAM은 텍스트 프롬프트로 원하는 물체만 골라 분할(segmentation)할 수 있는 모델이다. 모든 물체를 다 분할하는 SAM보다 훨씬 빠르다.

```python
model = FastSAM("FastSAM-s.pt")
results = model(source, texts="dog")   # "강아지"만 골라 분할
output_image = results[0].plot()
```

이미 만들어진 두 종류의 웹 UI도 실행 방식이 다르다는 점이 중요하다. `SearchApp`(이미지 검색)은 파이썬 스크립트로 바로 실행하고, `Inference`(실시간 대시보드)는 `streamlit run`으로 실행한다.

```python
# python으로 직접 실행
app = solutions.SearchApp(device="cpu")
app.run(debug=True)

# streamlit run 파일명.py 로 실행
inf = solutions.Inference(model="yolo11n.pt")
inf.inference()
```

**실무 예시**: FastSAM의 텍스트 프롬프트 분할은 배경 제거나 특정 객체 합성이 필요한 이미지 편집 툴에 빠르게 적용할 수 있고, 두 종류의 웹 UI는 사내 데모나 PoC 단계에서 별도 프론트엔드 코드 없이 결과를 빠르게 보여줘야 할 때 그대로 붙여 쓰는 패턴이다.

## 핵심 정리
- `solutions.*` 클래스는 `results[0].plot()`이 아니라 `results.plot_im`으로 이미 그려진 결과를 반환한다
- 좌표 개수가 로직을 바꾼다: 2점은 선(IN/OUT), 4점 이상은 다각형(영역 내부 카운팅)
- SAHI는 이미지를 타일로 쪼개 추론해 작은 물체 탐지율을 끌어올리는 기법이다
- 실시간 처리 속도 측정은 `time.perf_counter()`로, 배포 전 OpenVINO 변환으로 CPU 성능을 검증한다
- `meter_per_pixel`, 마우스 좌표 추출처럼 카메라/환경이 바뀔 때마다 다시 보정해야 하는 값들이 있다
