## 코딩 학습

### YOLO 4대 핵심 기능 구조

YOLO(You Only Look Once)의 분류(Classification)·탐지(Detection)·자세(Pose)·분할(Segmentation) 네 가지 기능은 코드로 보면 같은 4단계 구조를 공유한다. 다른 건 **모델 가중치 파일**뿐이다.

```python
from ultralytics import YOLO
import cv2

# 1. 모델 로드 (태스크별 가중치만 다름)
model = YOLO("가중치.pt")

# 2. 모델 추론
results = model("입력이미지.jpg")

# 3. 결과 시각화 (박스/점/외곽선을 이미지 위에 그려줌)
result_image = results[0].plot()

# 4. 결과 이미지 저장
cv2.imwrite("결과이미지.jpg", result_image)
```

`results[0].plot()` 한 줄이 태스크와 무관하게 "탐지 결과를 이미지 위에 그려서 반환"해주기 때문에, 모델만 바꾸면 다른 결과가 나온다.

| 기능 | 모델(`.pt`) | 입력 → 결과 | 그려지는 것 |
|---|---|---|---|
| 분류 | `yolo11n-cls.pt` | `input.jpg` → `result.jpg` | 클래스 이름 1개(텍스트만, 박스 없음) |
| 탐지 | `yolo11x.pt` | `input_det.jpg` → `result_det.jpg` | 물체마다 사각형 박스 + 클래스명 + 신뢰도 |
| 자세 | `yolo11n-pose.pt` | `input_pose.jpg` → `result_pose.jpg` | 관절 17개 점 + 뼈대 연결선 |
| 분할 | `yolo11n-seg.pt` | `input_seg.jpg` → `result_seg.jpg` | 물체 윤곽선을 따라 색칠된 마스크 |

분류는 위치 정보가 아예 없어서 "이 사진 전체가 뭔지" 하나만 답하고, 탐지는 사람이 크고 정확한 `yolo11x.pt`를 써서 물체마다 박스를 여러 개 그린다. 자세는 박스 대신 관절 좌표 17개를 예측해서 점과 뼈대선으로 표현하고, 분할은 유일하게 `model(..., save=True)`를 추가로 넘겨서 `runs/segment/predict/`에 자동 저장본이 하나 더 생긴다.

**실무 예시**: 어떤 기능을 쓸지는 "무엇을 판단해야 하는가"로 정해진다. 상품 하나만 있는 사진에서 카테고리를 알고 싶으면 분류, 매장 CCTV에서 사람·카트 위치를 알고 싶으면 탐지, 헬스장 자세 교정 앱이면 포즈, 자율주행에서 도로 경계를 정확히 따야 하면 분할을 쓴다.

### 원하는 클래스만 골라 탐지하기

탐지 결과에서 모든 클래스가 다 필요한 건 아니다. `model.names`로 COCO 80개 클래스 이름을 먼저 확인하고, `classes=[...]`로 원하는 클래스 번호만 골라 탐지하도록 필터링할 수 있다.

```python
model = YOLO("yolo11n.pt")
print(model.names)          # COCO 80개 클래스 이름 확인
model(
    "input_params2.jpg",
    classes=[60, 75],       # 60=식탁, 75=꽃병 등 원하는 클래스만
    save=True
)
```

실습 결과 폴더(`runs/detect/predict-4`)를 보면 `crops/person`, `crops/cup`, `crops/bowl`처럼 탐지된 물체만 잘라 저장하는 옵션, `predict-5/labels`처럼 좌표를 텍스트로 저장하는 옵션까지 실험한 흔적이 있다.

**실무 예시**: 매장 CCTV에서 "사람과 카트만" 세는 식으로 불필요한 클래스(의자, 화분 등)를 걸러내 오탐을 줄일 때 `classes` 필터링을 그대로 쓴다.

### 실시간 스트리밍 처리와 상태 판정

정지 이미지 추론에서 실시간 영상으로 넘어가면 `cv2.VideoCapture(stream_url)`로 CCTV의 `.m3u8`(HLS) 스트림 주소를 읽어 `while cap.isOpened()` 루프 안에서 프레임마다 추론한다. 여기에 탐지 개수를 세서 Safe/Warning 같은 상태를 판정해 화면에 표시하는 방식으로 응용할 수 있다.

```python
cap = cv2.VideoCapture("https://.../playlist.m3u8")  # 실제 스트림 주소
model = YOLO("yolo11n.pt")
WARNING_THRESHOLD = 5

while cap.isOpened():
    success, frame = cap.read()
    if not success:
        break

    results = model(frame)
    annotated_frame = results[0].plot()

    count = len(results[0].boxes)
    status = "Warning" if count >= WARNING_THRESHOLD else "Safe"
    color = (0, 0, 255) if status == "Warning" else (255, 0, 0)

    cv2.putText(annotated_frame, f"Detected : {count}, {status}",
                (10, 30), cv2.FONT_HERSHEY_SIMPLEX, 1, color, 2, cv2.LINE_AA)
    cv2.imshow("YOLO_COUNT", annotated_frame)

    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

cap.release()
cv2.destroyAllWindows()
```

**실무 예시**: 실제 CCTV 연동 서비스에서 "위험 인원 초과", "혼잡도 경고" 같은 알림 로직이 바로 이 `count` 기준 상태 판정 패턴으로 만들어진다.

### 객체 추적(Tracking)과 스트림 URL 오류

`model()` 대신 `model.track(frame, persist=True, conf=0.6)`을 쓰면 프레임마다 새로 탐지하는 게 아니라, 이전 프레임의 물체와 같은 물체면 같은 추적 ID를 유지한다. `persist=True`가 빠지면 매 프레임을 독립적으로 처리해서 같은 사람도 프레임마다 새 ID로 잘못 세게 된다.

```python
results = model.track(frame, persist=True, conf=0.6)
```

이 코드를 실습하다가 `cap = cv2.VideoCapture("https://www.earthcam.com/world/ireland/dublin/?cam=templebar")`처럼 브라우저용 웹페이지 주소를 그대로 넣어서 실행이 안 되는 문제를 만났다. `cv2.VideoCapture()`는 사람이 보는 HTML 페이지를 열 수 없고, ffmpeg 백엔드가 이해하는 `.m3u8`/`.mp4`/`rtsp://` 같은 **직접 스트림 주소**만 받을 수 있다. 그래서 `cap.isOpened()`가 열리지 않거나 `cap.read()`가 바로 실패해서 "프레임 읽기 실패" 분기로 빠지고 루프가 끝나버렸다. 실제 스트림 주소는 브라우저 개발자 도구(Network 탭)에서 `.m3u8` 요청을 찾아야 얻을 수 있다.

**실무 예시**: 매장 동선 분석에서 "같은 사람이 몇 개 구역을 돌았는지" 세려면 `persist=True` 추적이 필수다. 그리고 CCTV/IP카메라 연동 시 "이 주소가 페이지인지 순수 스트림 엔드포인트인지" 확인하는 습관은 실무에서 자주 겪는 실수를 줄여준다.

## 핵심 정리
- YOLO 분류/탐지/자세/분할은 파이프라인이 동일하고 모델 가중치 파일만 다르다
- `classes=[...]`로 원하는 클래스만 걸러 탐지 결과의 노이즈를 줄일 수 있다
- 실시간 스트리밍은 `while cap.isOpened()` 루프 + 탐지 개수 기반 상태 판정으로 응용된다
- `model.track(persist=True)`는 프레임 간 동일 물체에 같은 ID를 유지해준다
- `cv2.VideoCapture()`에는 웹페이지가 아니라 `.m3u8`/`rtsp` 같은 직접 스트림 주소만 넣을 수 있다
