## JavaScript, 앞에서부터 하나씩

JS를 처음 보면 괄호, 점, 화살표가 한꺼번에 쏟아져서 겁이 난다.

```js
btn.addEventListener("click", () => {
    out.textContent = `안녕, ${myName}!`
})
```

그런데 이걸 **앞에서부터 한 덩어리씩** 뜯어보면 생각보다 단순하다.

- `btn` — 버튼 요소
- `.addEventListener` — 이벤트 감지를 달겠다
- `"click"` — 클릭할 때
- `() => { }` — 그때 실행할 함수
- `out.textContent` — out 요소의 텍스트 내용
- `` `안녕, ${myName}!` `` — 변수를 끼워 넣은 문자열

처음부터 전체를 이해하려 하지 말고, **왼쪽에서 오른쪽으로, 위에서 아래로** 읽으면 된다.

## 코딩 학습

### JavaScript란

| | 역할 |
|--|--|
| HTML | 뼈대 |
| CSS | 옷과 색깔 |
| JavaScript | 움직임과 반응 |

JS로 할 수 있는 것:
- 버튼 클릭하면 글자·그림 바꾸기
- 사용자가 입력한 값 읽기
- 계산하고 결과 화면에 보여주기
- 조건에 따라 다른 동작 실행하기

JS는 **위에서 아래로 순서대로 읽힌다.** 그래서 `<script>` 태그는 반드시 `</body>` 바로 위에 놔야 한다.

```html
<body>
  <h1 id="title">제목</h1>

  <script src="scripts/main.js"></script>  <!-- 항상 맨 아래 -->
</body>
```

`<head>`에 두면 HTML이 다 그려지기 전에 JS가 실행돼서 요소를 찾지 못하고 오류가 난다.

### 변수 let과 const

**`let`** — "새 변수(상자) 만들게"

```javascript
let score = 0;
```

- `let` : 새 변수를 만드는 키워드
- `=` : "이 값을 상자에 넣어"라는 뜻. **같다는 비교가 아님**
- `score` : 내가 붙인 상자 이름
- `0` : 상자에 넣을 초기값

**`let` vs `const`**

```javascript
let score = 0;         // 나중에 바꿀 수 있는 상자
score = score + 1;     // 가능 ✅

const btn = document.querySelector("#btn");  // 한 번 담으면 고정
btn = null;            // 오류 ❌
```

| | 재할당 | 주로 쓰는 경우 |
|--|--|--|
| `let` | 가능 | 카운터, 계산 결과 등 바뀌는 값 |
| `const` | 불가 | HTML 요소 참조 등 안 바뀌는 값 |

**`.value`** — 입력칸에 친 글자를 꺼내는 속성

```javascript
const input = document.querySelector("#name")
let myName = input.value;  // input에 입력된 텍스트를 꺼냄
```

**템플릿 리터럴** — 변수를 문자열 안에 끼워 넣는 방법

```javascript
`안녕, ${myName}!`       // ✅ 백틱 + ${} 사용
"안녕," + myName + "!"   // 옛날 방식
```

백틱(`` ` ``)은 키보드 숫자 `1` 왼쪽 키다. 작은따옴표(`'`)와 헷갈리지 말 것.

### DOM 조작

HTML 요소를 JS에서 다루려면 먼저 가져와야 한다.

```javascript
const title = document.querySelector("#title")
```

HTML의 `id`는 HTML/CSS용 이름표고, `const title`은 JS 안에서 쓰는 이름표다.
태그 자체가 `title`이 되는 게 아니라, `title`이 그 태그를 **가리키는** 것이다.

| 선택자 | 의미 |
|--|--|
| `"#title"` | id가 title인 요소 |
| `".box"` | class가 box인 요소 |
| `"h1"` | h1 태그 |

요소를 가져온 뒤엔 내용을 바꿀 수 있다.

```javascript
title.textContent = "hello"   // 텍스트만 바꿈 (권장)
title.innerHTML = "<b>hello"  // HTML 태그까지 삽입 가능
```

### 이벤트

```javascript
btn.addEventListener("click", () => {
    title.textContent = "hello world"
})
```

`addEventListener`는 인자를 2개 받는다.

1. **어떤 이벤트인지** — `"click"`
2. **그때 뭘 할지** — `() => { ... }`

`()` 안은 보통 비워둔다. 클릭 정보(좌표, 대상 요소 등)가 필요할 때만 `(e)`를 넣는다.

```javascript
// e가 필요한 경우 — 링크 기본 이동 막기
link.addEventListener("click", (e) => {
    e.preventDefault()  // 페이지 이동 차단
})
```

`e`는 관례상 쓰는 이름이고 `event`, `ev` 아무 이름이나 써도 된다.

### 조건문

```javascript
if (op.value === "+") { result = x + y; }
else if (op.value === "-") { result = x - y; }
else if (op.value === "*") { result = x * y; }
else { result = x / y; }
```

- `===` — 값과 타입이 같은지 비교. `=`(대입)과 전혀 다름
- `select` 태그에서 선택된 값은 `option`의 `value`로 전달됨

```html
<!-- value가 있어야 JS에서 읽힘 -->
<option value="choco">초코</option>
<option value="vanilla">바닐라</option>
```

`id`로 비교하면 작동하지 않는다. 반드시 `value`를 써야 한다.

### 함수

```javascript
const multi = (n1, n2) => n1 * n2;
```

화살표 함수로 선언하고 `const`로 저장해서 재사용한다.
한 줄이면 `{}`와 `return` 생략 가능하다.

```javascript
// 인자 여러 개
const calc = (n1, n2, n3) => n1 * n2 + n3;

// 호출
calc(2, 3, 4)  // → 10
```

## 자주 틀린 것들

| 틀린 코드 | 올바른 코드 | 이유 |
|--|--|--|
| `a,value` | `a.value` | 쉼표 → 점 |
| `'${myName}'` | `` `${myName}` `` | 작은따옴표 → 백틱 |
| `.addEventListener=("click"` | `.addEventListener("click"` | `=` 빼야 함 |
| `querySelector(#id)` | `querySelector("#id")` | 따옴표 필수 |
| `option id`로 비교 | `option value`로 비교 | select는 value로 읽음 |
| `Number(a,value)` | `Number(a.value)` | 쉼표 → 점 |

공통된 실수 패턴: **점(`.`)을 쉼표(`,`)로 오타**, **따옴표 종류 혼동**

## 핵심 정리

- JS는 위에서 아래로 읽힌다. 코드도, `<script>` 위치도 순서가 중요
- `=`는 대입, `===`는 비교. 절대 헷갈리지 말 것
- `let`은 바뀌는 값, `const`는 고정값. 요소 참조는 항상 `const`
- 템플릿 리터럴(백틱 + `${}`)로 변수를 문자열에 끼워 넣는다
- 한꺼번에 이해하려 하지 말고, 앞에서부터 한 덩어리씩 읽으면 된다
