## 코딩 학습

### React란 무엇인가

React는 화면을 **컴포넌트** 단위로 쪼개서 만드는 JavaScript 라이브러리다. HTML을 처음부터 문자열로 조립하는 대신, `<Card/>`, `<Avatar/>`처럼 재사용 가능한 조각을 만들고 그 조각들을 조합해서 페이지를 완성한다.

핵심 특징 두 가지:
- **선언형(Declarative)**: "이 버튼을 클릭하면 이 태그를 지워라" 식으로 DOM을 직접 조작하지 않고, "이 상태일 때 화면은 이렇게 생겨야 한다"를 선언하면 React가 알아서 실제 DOM을 갱신해준다.
- **props와 state**: 컴포넌트 밖에서 값을 넣어주는 `props`(읽기 전용), 컴포넌트 내부에서 스스로 바꾸는 `state`(`useState`)로 데이터를 관리한다.

JSX는 이 컴포넌트 안에서 JavaScript와 HTML 비슷한 마크업을 같이 쓸 수 있게 해주는 문법이다.

```jsx
const Greeting = ({name}) => {
    return <h1>안녕, {name}</h1>   // {} 안에는 JS 표현식이 그대로 들어감
}
```

### main.jsx → App.jsx → 컴포넌트 폴더, 연결 구조

오늘 가장 헷갈렸던 부분은 "결국 뭐가 화면에 뜨는 기준이냐"였다. 실제 흐름은 아래처럼 위에서 아래로 이어진다.

```
index.html
  <div id="root"></div>
  <script type="module" src="/src/main.jsx"></script>
        │
        ▼
main.jsx                         ← 진짜 진입점(entry point)
  import './index.css'           ← 전역 스타일, 앱 전체에 항상 적용
  createRoot(#root).render(<App/>)
        │
        ▼
App.jsx                          ← 화면의 뼈대, 컴포넌트를 불러와 배치만 함
  import Card from './components/21/Card.jsx'
  import Avatar from './components/22/Avatar.jsx'
  <Card .../>  <Avatar .../>
        │
        ▼
components/20/Profile.jsx
components/21/Card.jsx  + Card.css
components/22/Avatar.jsx + Avatar.css
components/23/Badge.jsx
...
```

정리하면 세 계층이다.

| 계층 | 역할 |
|---|---|
| `index.html` | `#root`라는 빈 껍데기와 `main.jsx`를 불러오는 스크립트 태그만 존재 |
| `main.jsx` | 전역 CSS를 불러오고, `<App/>`을 `#root`에 그려 넣는 유일한 진입점 |
| `App.jsx` | 컴포넌트들을 import해서 화면을 조립하는 "설계도" |
| `components/N/*.jsx` | 각자 자기 역할 하나만 하는 부품. 필요하면 같은 폴더에 CSS 파일을 같이 둠 |

`App.jsx`가 아무리 바뀌어도 `main.jsx`는 거의 안 건드리는 이유가 여기 있다 — 진입점은 고정해두고, 그 밑(App과 컴포넌트들)만 계속 갈아끼우는 구조이기 때문이다. 그리고 컴포넌트 CSS(`Card.css`, `Avatar.css`)는 해당 컴포넌트를 쓸 때만 적용되지만, `index.css`는 `main.jsx`가 항상 불러오므로 어떤 컴포넌트를 켜고 끄든 상관없이 전역으로 깔려 있다.

### 실전 조합 예시

React 컴포넌트는 원래 서로 안에 끼워 넣는(nesting) 방식으로 조합해서 쓴다. 각 컴포넌트가 실제 사이트에서 어디 쓰이는지, 그리고 여러 개를 합치면 어떤 화면이 되는지 정리했다. 아래 세 데모는 실제로 클릭이 동작하는 버전을 따로 만들어뒀다 → [React 컴포넌트 조합 쇼케이스](https://claude.ai/code/artifact/0736c1ad-62c9-456f-ab39-08fa077c9107)

배열을 `map`으로 순회하는 건 아직 안 배워서, 항목이 여러 개인 예시(채팅 목록, 상품 그리드)도 컴포넌트를 하나씩 직접 나열하는 방식으로만 만들었다. 대신 "어떤 값을 넘겼는지(App.jsx)"와 "그 값을 컴포넌트가 내부에서 어떻게 쓰는지"를 나눠서 봤다.

| 컴포넌트 | 실제 쓰이는 곳 |
|---|---|
| `Hello` | 랜딩 페이지 히어로 문구, 온보딩 첫 화면 |
| `Greeting` | 로그인 후 "안녕하세요, OO님" 대시보드 헤더 |
| `Profile` | 팀 소개 페이지, LinkedIn 스타일 프로필 카드 |
| `Card` | 서비스 소개 그리드, 요금제 카드 |
| `Avatar` | 채팅 앱 친구 목록의 온라인 표시, 화상회의 참가자 목록 |
| `Badge` | 쇼핑몰 "NEW"/"품절" 태그, 주문 상태 라벨 |

**① 팀 페이지 멤버 카드** — `Card`(틀) 안에 `Avatar` + `Badge`를 같이 배치, 온라인 사각형을 누르면 즉시 토글된다.

App.jsx — 입력값:
```jsx
<div className="card">
    <Avatar name="김해냄" online={true} />
    <Badge text="팀장" type="new" />
</div>
```

components/22/Avatar.jsx — 내부 로직 (`online={true}`가 `useState`의 시작값이 됨):
```jsx
const Avatar = ({name, online}) => {
    const [isOnline, setIsOnline] = useState(online)

    return (
        <button
            className={isOnline ? 'online' : 'offline'}
            onClick={() => setIsOnline(!isOnline)}
        />
    )
}
```

이름 + 온라인 여부 + 직급 라벨이 한 카드에 다 들어간 팀원 소개 UI가 된다. GitHub Contributors 목록, Slack 멤버 리스트가 이 구조다.

**② 채팅 앱 사이드바** — `Greeting` + `Avatar` 두 개를 각각 직접 나열, 대화를 클릭하면 읽음 처리된다.

App.jsx — 입력값(초기 unread 개수):
```jsx
const [unread1, setUnread1] = useState(3)  // 김냄냄
const [unread2, setUnread2] = useState(1)  // 박모카
```

App.jsx — 화면(친구마다 직접 나열):
```jsx
<Greeting name="해냄" />

<div onClick={() => setUnread1(0)}>
    <Avatar name="김냄냄" online={true} />
    {unread1 > 0 && <span>{unread1}</span>}
</div>

<div onClick={() => setUnread2(0)}>
    <Avatar name="박모카" online={false} />
    {unread2 > 0 && <span>{unread2}</span>}
</div>
```

상단엔 개인화 인사, 그 아래 친구 목록엔 온라인 여부, 옆에는 안 읽은 메시지 뱃지가 `&&` 조건부 렌더링으로 붙는다. 클릭하면 `unread`가 0이 되면서 `unread1 > 0 && ...` 조건이 거짓이 되어 뱃지가 사라진다. 카카오톡/인스타 DM 목록이 정확히 이 패턴이다.

**③ 상품 카드 그리드** — `Card` + `Badge`, 카드를 클릭하면 삼항 연산자로 색이 전환된다.

App.jsx — 입력값(카드별 초기 type):
```jsx
const [type1, setType1] = useState('sale')  // 헤드폰
const [type2, setType2] = useState('new')   // 키보드
const [type3, setType3] = useState('sale')  // 백팩
```

App.jsx — 화면(카드별로 직접 나열, type2·type3 카드도 동일하게 반복):
```jsx
<div onClick={() => setType1(type1 === 'new' ? 'sale' : 'new')}>
    <Badge text={type1 === 'new' ? 'NEW' : 'SALE'} type={type1} />
    <Card emoji="🎧" title="헤드폰" desc="노이즈캔슬링" />
</div>
```

components/23/Badge.jsx — 내부 로직:
```jsx
const Badge = ({text, type}) => {
    const color = type === "new" ? 'green' : 'crimson'
    return <span style={{backgroundColor: color}}>{text}</span>
}
```

카드 자체는 재사용하고 그 위에 `Badge`만 조건부로 얹으면, 같은 `Card`/`Badge` 조합으로 "일반 카드"와 "할인 중인 카드"를 구분해서 그릴 수 있다.

## 핵심 정리
- React는 화면을 컴포넌트 단위로 쪼개고, props(외부 입력)와 state(내부 상태)로 데이터를 관리하는 라이브러리
- 진입점은 `main.jsx` 하나로 고정, `App.jsx`와 `components/*`만 계속 갈아끼우는 구조
- `index.css`(전역)와 컴포넌트별 CSS(`Card.css` 등)는 적용 범위가 다르다 — 전역은 항상 살아있고, 컴포넌트 CSS는 그 컴포넌트를 쓸 때만 적용
- 오늘 만든 6개 컴포넌트(Hello~Badge)는 각각 "고정 텍스트 → props 받기 → 기본값 → 여러 props 조합 → 상태 관리 → 조건부 스타일" 순으로 난이도가 올라가는 구성
- 실제 서비스에서는 이 작은 컴포넌트들을 여러 개 조합(nesting)해서 팀 페이지, 채팅 사이드바, 상품 그리드 같은 화면을 완성한다
