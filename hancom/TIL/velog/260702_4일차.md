### 1-1. CLAUDE.md로 Claude에게 규칙 기억시키기

Claude Code는 대화 시작 시 프로젝트 안의 `CLAUDE.md` 파일을 자동으로 읽는다. 다른 이름의 파일은 무시되기 때문에, 자동화 규칙이나 반복 명령어를 기억시키려면 반드시 `CLAUDE.md` 안에 있어야 한다.

내용이 많아지면 `@파일명` 문법으로 분리할 수 있다.

> `@파일명`을 쓰면 Claude가 그 파일을 이어서 읽는다.  
> 실질적으로 한 파일에 다 쓴 것과 같은 효과.  
> 새 자동화 규칙이 생기면 파일 하나 만들고 CLAUDE.md에 한 줄 추가하면 끝.

```
# CLAUDE.md 예시

@claude_css_folder.md   ← CSS 폴더 생성 자동화 규칙
@claude_til.md          ← TIL 저장 자동화 규칙
@claude_velog.md        ← Velog 포스트 생성 규칙
```

### 2-1. CSS position 속성 4종

`position` 속성은 요소를 어디에, 무엇을 기준으로 배치할지 결정한다.

| 값 | 기준점 | 원래 자리 | 주요 용도 |
|---|---|---|---|
| `relative` | 원래 자리 | 유지 | 자식의 기준점 잡기 |
| `absolute` | 가장 가까운 positioned 부모 | 포기 | 특정 위치에 겹치기 |
| `fixed` | 브라우저 화면(viewport) | 포기 | 네비바, 플로팅 버튼 |
| `sticky` | 부모 요소 | 유지 | 섹션 제목 고정 |

**relative + absolute 세트 패턴**

> `absolute`는 positioned 부모를 기준으로 위치를 잡는다.  
> 부모에 `relative`를 주면 자식의 기준점 역할을 하게 된다.  
> 이 세트를 함께 쓰는 게 가장 흔한 패턴.

```css
.parent { position: relative; }            /* 기준점 역할 */
.child  { position: absolute;
           bottom: 10px; right: 16px; }   /* 부모 기준으로 위치 계산 */
```

**fixed vs sticky**

> `fixed` — 스크롤과 무관하게 화면에 고정. 플로팅 버튼에 사용.  
> `sticky` — 스크롤 중 지정 위치에 닿으면 붙었다가, 부모를 벗어나면 사라짐.

```css
.floating-btn  { position: fixed;  bottom: 24px; right: 24px; } /* 항상 화면 고정 */
.section-title { position: sticky; top: 0; }                     /* 부모 범위 안에서만 고정 */
```

### 2-2. CSS 네온 효과

**box-shadow로 빛 번짐 만들기**

> `box-shadow`를 여러 겹 쌓으면 빛이 번지는 네온 느낌이 난다.  
> 주의: `rgba()` 안에 `rgb()`를 중첩하면 브라우저가 파싱을 못 한다.

```css
.neon {
    box-shadow:
        0 0 10px rgba(0, 255, 234, 0.7),  /* 가장 밝은 중심 */
        0 0 25px rgba(0, 255, 234, 0.5),  /* 중간 번짐 */
        0 0 50px rgba(0, 255, 234, 0.3);  /* 바깥쪽 옅은 번짐 */
}
```

**폰트 그라데이션**

> CSS에는 텍스트에 직접 그라데이션을 입히는 속성이 없다.  
> 배경에 그라데이션을 깔고 글자 모양으로 잘라내는 우회 방식을 쓴다.  
> `display: inline-block` 필수 — 없으면 마지막 글자가 잘릴 수 있다.

```css
p {
    display: inline-block;
    background: linear-gradient(90deg, #00ffea, #5200ff);
    -webkit-background-clip: text;    /* 배경을 글자 모양으로 자름 */
    -webkit-text-fill-color: transparent; /* 글자를 투명하게 → 배경이 비쳐 보임 */
    background-clip: text;
}
```

### 2-3. flex vs text-align — 가운데 정렬의 차이

| | `text-align: center` | `display: flex` + align/justify |
|---|---|---|
| 정렬 대상 | 텍스트(인라인 요소) | 자식 요소 전체 |
| 정렬 축 | 가로만 | 가로 + 세로 동시 |
| 적용 위치 | 요소 자신 | 부모 |

**absolute 구조에서 flex가 필요한 이유**

> 카드 앞·뒷면처럼 두 요소를 같은 자리에 겹쳐야 할 때 `position: absolute`를 쓴다.  
> 이때 요소가 높이를 잃기 때문에 `text-align`만으론 세로 가운데를 잡을 수 없다.  
> 겹치려고 `absolute` → 높이를 잃음 → 세로 정렬을 위해 `flex`가 따라오는 것.

```css
.front, .back {
    position: absolute;
    inset: 0;                        /* 부모를 꽉 채움 → 두 요소 겹침 */
    display: flex;
    align-items: center;             /* 세로 가운데 */
    justify-content: center;         /* 가로 가운데 */
    backface-visibility: hidden;     /* 뒤집혔을 때 안 보이게 */
}
.back { transform: rotateY(180deg); } /* 뒷면은 처음부터 뒤집혀 있음 */
```

**두 방식 차이를 눈으로 확인하는 방법**

> `height`를 충분히 줘야 세로 방향 차이가 눈에 보인다.

```css
.text-card { text-align: center; }                                       /* 글자 상단에 붙음 */
.flex-card { display: flex; align-items: center; justify-content: center; } /* 글자 정중앙 */
.card      { height: 120px; }
```

## 핵심 정리

- `CLAUDE.md`는 Claude Code가 자동으로 읽는 파일. `@파일명`으로 기능별 분리 관리 가능
- `relative + absolute` 세트: relative가 absolute 자식의 기준점 역할
- `fixed`는 화면 고정, `sticky`는 부모 범위 안에서만 고정
- 폰트 그라데이션은 배경 그라데이션 + 클리핑 우회 방식
- `text-align`은 가로만, `flex`는 가로+세로 — `absolute` 구조에서는 flex가 필수
