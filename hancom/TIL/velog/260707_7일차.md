## 코딩 학습

### DOM 조작 심화

오늘 가장 많이 헷갈렸던 것은 `querySelector`와 `createElement`의 차이다.

```js
const li = document.querySelector("li")   // ❌ 기존 DOM에서 찾기 → 없으면 null
const li = document.createElement("li")   // ✅ 새 요소 생성
```

`querySelector`는 **찾는** 것이고, `createElement`는 **만드는** 것이다. 만든 요소는 `append` 또는 `appendChild`로 붙여야 화면에 나타난다.

```js
const li = document.createElement("li")
li.textContent = "사과"
out.appendChild(li)   // 이 줄이 없으면 화면에 안 보임
```

**innerHTML vs textContent**

| | `innerHTML` | `textContent` |
|---|---|---|
| HTML 태그 | 해석해서 렌더링 | 문자 그대로 출력 |
| 용도 | `<br>` 등 태그 포함 | 텍스트만 바꿀 때 |

```js
el.innerHTML = "줄1<br>줄2"   // 줄바꿈 렌더링
el.textContent = "줄1<br>줄2" // <br> 그대로 출력
```

**`innerHTML = ""`을 쓰는 이유**

`appendChild`는 추가만 하기 때문에 버튼을 여러 번 누르면 목록이 누적된다. 클릭할 때마다 먼저 비우고 새로 채워야 한다.

```js
document.querySelector("#run").addEventListener("click", () => {
    out.innerHTML = ""   // 먼저 비우고
    for (let i = 1; i <= count; i++) {
        const li = document.createElement("li")
        li.textContent = `${i}번째`
        out.appendChild(li)
    }
})
```

**render() 패턴**

화면 업데이트 코드를 함수로 묶으면 버튼마다 반복 작성하지 않아도 된다.

```js
const render = () => {
    out.textContent = fruits.join(", ")
    info.textContent = `개수: ${fruits.length}`
}
render()  // 페이지 로드 시 초기값 표시

document.querySelector("#add").addEventListener("click", () => {
    fruits.push(fruit.value)
    render()   // 한 줄로 끝
})
document.querySelector("#del").addEventListener("click", () => {
    fruits.pop()
    render()
})
```

### 반복문 for vs while

```js
// for — 횟수가 정해진 경우
for (let i = 1; i <= count; i++) { ... }

// while — 최고점에서 시작해서 내려오는 경우
let i = Number(n.value)
while (i > 0) { i-- }
```

대부분 서로 바꿔 쓸 수 있다. `for`는 카운터 변수가 괄호 안에 모여 있어서 읽기 편하고, `while`은 끝 시점이 불확실하거나 카운트다운처럼 최고점에서 시작할 때 자연스럽다.

`while` 안에서 변화값(`i--`) 빠뜨리면 무한루프 주의.

### 배열 메서드

```js
let fruits = ["사과", "바나나"]

fruits.push("망고")              // 끝에 추가 → ["사과", "바나나", "망고"]
fruits.pop()                     // 끝에서 제거 → ["사과", "바나나"]
fruits.filter(f => f !== "사과") // 조건에 맞는 것만 남김 → ["바나나"]
fruits.join(", ")                // 문자열로 이어붙임 → "사과, 바나나"
```

`pop()`은 인자를 넣어도 무시하고 무조건 마지막 요소를 제거한다. 특정 항목을 삭제하려면 `filter`를 사용해야 한다.

```js
// "망고" 삭제
fruits = fruits.filter(f => f !== "망고")
```

### 클래스

객체를 찍어내는 틀이다. 강아지가 100마리여도 클래스 하나로 관리할 수 있다.

```js
class Dog {
    constructor(name, age) {   // new로 만들 때 자동 호출
        this.name = name       // 객체에 저장 (함수 끝나도 유지)
        this.age = age
    }
    bark() {
        return `${this.name}: 멍멍!`
    }
    info() {
        return `(${this.age}살)`
    }
}

const poppy = new Dog("뽀삐", 3)
const choco = new Dog("초코", 5)

out.textContent = `${poppy.bark()}${poppy.info()} ${choco.bark()}${choco.info()}`
// 뽀삐: 멍멍!(3살) 초코: 멍멍!(5살)
```

| 키워드 | 역할 |
|---|---|
| `class` | 틀(설계도) 선언 |
| `constructor` | 객체 생성 시 자동 실행, 이름 못 바꿈 |
| `this` | 지금 만들어지는 객체 자신 |
| `new` | 틀로 객체를 찍어내는 키워드 |

주의: 속성명과 메서드명이 같으면 충돌한다.

```js
this.age = age   // 속성 "age"
age() { ... }    // 메서드 "age" → 속성 덮어씀 ❌
getAge() { ... } // 이름 다르게 → 정상 ✅
```

### 서버 연동 fetch + json-server

**json-server**

`db.json` 파일 하나로 자동으로 CRUD API를 만들어주는 도구. 서버 코드 없이 연습용 백엔드로 사용한다.

```
npx json-server@0.17.4 db.json --static .
→ http://localhost:3000 에서 서버 + 정적 파일 동시 서빙
```

**fetch + async/await**

```js
// fetch — 서버에 요청 보내는 함수
// async/await — 응답 올 때까지 기다리게 하는 키워드 (항상 세트)

async () => {
    const res = await fetch("/users")    // 요청 보내고 기다림
    const users = await res.json()       // 응답을 JS 객체로 변환
}
```

`await` 없으면 응답 안 기다리고 다음 줄로 넘어가버린다.

**CRUD 요청 방식**

```js
// GET (기본값)
fetch("/users")

// POST — 추가
fetch("/users", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ name: "John" })
})

// PUT — 수정 (특정 id)
fetch("/users/1", {
    method: "PUT",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ name: "Jane" })
})

// DELETE — 삭제
fetch("/users/1", { method: "DELETE" })
```

## 핵심 정리
- `querySelector`는 찾기, `createElement`는 만들기 — 헷갈리면 오류남
- `innerHTML = ""`으로 먼저 비우고 채워야 누적 안 됨
- `render()` 함수로 화면 업데이트를 하나로 묶으면 코드가 훨씬 깔끔해짐
- 클래스는 `class` + `constructor` + `this` + `new` 네 가지 세트로 외우기
- `fetch`는 서버 요청, `await`는 기다리기 — 반드시 `async` 함수 안에서 사용
