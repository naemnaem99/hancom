React 컴포넌트 조합 · 실전 데모

작은 컴포넌트 6개가 어떻게 진짜 화면이 되는가

Card · Avatar · Badge · Greeting을 따로 만들었던 이유는 결국 이렇게 서로 안에 끼워 넣기 위해서다. 아래 세 데모는 실제로 클릭이 동작한다 — 오늘까지 배운 useState 토글과 && 조건부 렌더링만으로 만들었다. 배열을 map으로 순회하는 건 아직 배우지 않아서, 컴포넌트를 하나씩 직접 나열하는 방식 그대로 옮겼다.

① 팀 페이지 멤버 카드

Card 안에 Avatar + Badge를 배치 — 온라인 점을 눌러보세요

useState 토글

App.jsx — 입력값

<div className="card">
  <Avatar name="김해냄" online={true} />
  <Badge text="팀장" type="new" />
</div>

components/22/Avatar.jsx — 내부 로직

const Avatar = ({name, online}) => {
  const [isOnline, setIsOnline] = useState(online)
  // online={true}가 useState의 시작값이 됨

  return (
    <button
      className={isOnline ? 'online' : 'offline'}
      onClick={() => setIsOnline(!isOnline)}
    />
  )
}

② 채팅 앱 사이드바

Greeting + Avatar 두 개를 각각 직접 나열 — 대화를 클릭하면 읽음 처리

useState + && 조건부 렌더링

App.jsx — 입력값(초기 unread 개수)

const [unread1, setUnread1] = useState(3)  // 김냄냄
const [unread2, setUnread2] = useState(1)  // 박모카

App.jsx — 화면(친구마다 직접 나열, map 아직 안 배움)

<Greeting name="해냄" />

<div onClick={() => setUnread1(0)}>
  <Avatar name="김냄냄" online={true} />
  {unread1 > 0 && <span>{unread1}</span>}
</div>

<div onClick={() => setUnread2(0)}>
  <Avatar name="박모카" online={false} />
  {unread2 > 0 && <span>{unread2}</span>}
</div>

③ 상품 카드 그리드

Card 3개를 직접 나열, 각 Badge는 클릭할 때마다 삼항 연산자로 색이 바뀜

useState + 삼항 연산자

App.jsx — 입력값(카드별 초기 type)

const [type1, setType1] = useState('sale')  // 헤드폰
const [type2, setType2] = useState('new')   // 키보드
const [type3, setType3] = useState('sale')  // 백팩

App.jsx — 화면(카드별로 직접 나열)

<div onClick={() => setType1(type1==='new'?'sale':'new')}>
  <Badge text={type1==='new'?'NEW':'SALE'} type={type1} />
  <Card emoji="🎧" title="헤드폰" desc="노이즈캔슬링" />
</div>
// type2, type3 카드도 동일하게 반복

components/23/Badge.jsx — 내부 로직

const Badge = ({text, type}) => {
  const color = type === "new" ? 'green' : 'crimson'
  return <span style={{backgroundColor: color}}>{text}</span>
}
