# Velog 포스트 자동생성

**트리거**: "N일차 Velog 포스트 만들어줘" / "TIL 파일들로 포스트 만들어줘"

## 동작 순서
1. 사용자가 소스 TIL 파일 지정 (파일명 또는 날짜)
2. `hancom/hancom/TIL/` 안에서 해당 파일들을 읽어 내용 파악
3. 아래 포맷으로 Velog 포스트 재구성
4. `hancom/hancom/TIL/velog/` 에 저장

## 저장 파일명
- 형식: `YYMMDD_N일차.md`
- 예시: `260702_3일차.md`

## 포스트 포맷

```markdown
# N일차 — 주제1 · 주제2 · 주제3

> 오늘 다룬 것: 주제1 / 주제2 / 주제3

## 목차
- [도구 & 환경](#도구--환경)
  - [세부항목](#세부항목)
- [코딩 학습](#코딩-학습)
  - [세부항목](#세부항목-1)

---

## 도구 & 환경

### 세부항목

설명...

---

## 코딩 학습

### 세부항목

설명...

```언어
코드 예시
```

| 항목 | 설명 |
|---|---|
| A | ... |

---

## 핵심 정리
- 오늘 배운 것 1
- 오늘 배운 것 2
```

## 포맷 규칙

**제목**: `N일차 — 주제1 · 주제2` (TIL 파일들의 핵심 주제 나열)

**큰 목차 2개 고정**:
- `도구 & 환경`: Claude, VS Code, 개발자도구, 설치/설정 관련 내용
- `코딩 학습`: HTML, CSS, JS 등 언어/코드 관련 내용
- 해당 카테고리 내용이 없으면 그 섹션 생략

**섹션 구성**:
- 각 큰 섹션 안에 `###` 소제목으로 세부 내용 구분
- 섹션 사이 `---` 수평선 필수
- 각 소제목에 코드블록 또는 표 하나 이상 포함

**마지막**: 핵심 정리 bullet 3~5개로 마무리

## TIL → Velog 변환 원칙
- TIL의 Q&A는 설명문 형태로 재작성 (블로그 독자 기준)
- 코드 예시는 그대로 유지
- 표는 재활용 또는 재구성
- 너무 세부적인 내용은 요약, 핵심만 남김
- 7~10분 읽기 분량 목표
