# 커밋 메시지 규칙

**형식**: `타입: 한글 설명`

타입은 영어, 설명은 한글로 통일. 지금까지 영어/한글이 섞여 있던 커밋 메시지를 이 규칙으로 일원화.

## 타입 목록

| 타입 | 의미 | 예시 |
|---|---|---|
| `add` | 새 파일/폴더/기능 추가 | `add: 13_yolo YOLO 추적 실습 코드 추가` |
| `fix` | 버그/오류 수정 | `fix: 18_ocr 이미지 경로 오류 수정` |
| `update` | 기존 내용 보완·개선 (새 기능 아님) | `update: OCR 스크립트에 전처리 로직 반영` |
| `remove` | 파일/폴더 삭제 | `remove: 07_groq 독립 저장소로 이전에 따라 삭제` |
| `docs` | TIL·velog·README 등 문서 작업 | `docs: 17일차 TIL·velog 정리` |
| `chore` | 설정 파일, gitignore, 폴더 구조 정리 등 | `chore: .gitignore에 임시 산출물 추가` |
| `refactor` | 동작 변화 없이 구조만 정리 | `refactor: 05_react 컴포넌트 폴더 구조 재정리` |

## 작성 규칙

- 제목 줄: `타입: 설명` 한 줄, 간결하게 (무엇을 했는지)
- 여러 폴더/주제를 한 커밋에 묶을 땐 `,`로 나열: `add: 15_openapi, 16_huggingface 실습 코드 추가`
- 이유(왜)가 코드만 봐서 명확하지 않을 때만 본문에 한 줄 추가 (예: 왜 이 방식을 선택했는지, 어떤 문제를 해결한 건지)
- 폴더 번호(`13_yolo`, `18_ocr` 등)를 설명 앞에 붙이면 나중에 찾기 쉬움

## 적용 범위

이 저장소(`hancom`)에서 생성하는 모든 커밋에 적용. 과거 커밋(영어/한글 혼용, `css 페이지 작업` 같은 불명확한 메시지)은 그대로 두고, **이 시점 이후 커밋부터** 적용.
